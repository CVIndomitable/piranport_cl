# v0.0.11-alpha — NPC System (深海 & 舰娘)

> **代号**: Fleet  
> **前置版本**: v0.0.10-alpha (Arsenal)  
> **目标**: 实现策划案第7章 NPC系统，包括深海敌人AI体系和舰娘友方NPC

---

## 背景与现状

### 已有基础
- `LowTierDestroyerEntity.java` — 副本用低级驱逐，已有基础AI框架
- `AircraftEntity.java` — 状态机模式参考（LAUNCHING→CRUISING→ATTACKING→RETURNING→REMOVED）
- `FireControlManager` — 火控锁定系统
- 完整的武器系统（炮击/雷击/航空/导弹/深弹）
- 副本系统（实例化战斗场、节点战斗、敌人集数据）

### 策划要求摘要
- **深海**: 集群共享警戒、待机闲逛、交战行为（环绕射击）、多种攻击弹道（抛物线/追踪/近炸）
- **舰娘**: 友方NPC（策划案未详细展开，本版本做基础框架）

## 开发任务清单

### Phase 1：深海基类与集群系统

#### 1.1 AbstractDeepOceanEntity 基类
- [ ] 继承 `Mob`，实现水上行走（与玩家变身逻辑类似）
- [ ] 属性系统：血量、护甲、航速、索敌范围、射程、武器类型列表
- [ ] 掉落表：经验、材料、装备（`LootTable` JSON驱动）
- [ ] 死亡动画：沉没效果（逐渐下沉+粒子）
- [ ] 实现 `addAdditionalSaveData` / `readAdditionalSaveData` 持久化集群ID
- [ ] 基类注册 `Attributes`：`MAX_HEALTH`, `MOVEMENT_SPEED`, `ARMOR`, `FOLLOW_RANGE`, `ATTACK_DAMAGE`

#### 1.2 集群系统 FleetGroup
- [ ] `FleetGroup`：成员列表（UUID集合）、共享目标、警戒状态枚举（IDLE / ALERT / COMBAT）
- [ ] `FleetGroupManager extends SavedData`：管理所有活跃集群
  - 生成深海时自动分配集群（根据生成点proximity或结构标记）
  - 成员死亡/卸载时自动移除
  - 全集群成员死亡时销毁集群
- [ ] **集群警戒传播逻辑**：
  - 任一成员发现目标 → 集群所有IDLE成员共享该目标进入COMBAT
  - 已在COMBAT的成员不会因集群发现新目标而切换目标
  - 被攻击的成员将攻击者作为目标（优先级高于集群共享目标）

#### 1.3 生成规则
- [ ] 主世界海洋群系自然生成（少量，用于前哨站/补给站周围）
- [ ] 副本维度由 `EnemySetData` 驱动生成
- [ ] 生成时自动组队到同一 `FleetGroup`
- [ ] 使用 `SpawnPlacementRegister` 限制只在水面/水中生成

### Phase 2：AI行为系统

#### 2.1 待机行为 IdleWanderGoal
- [ ] 继承 `RandomStrollGoal`，限制在水面移动
- [ ] 在生成点附近闲逛（半径可配置，默认32格）
- [ ] 走走停停的自然节奏（与原版生物一致）

#### 2.2 交战行为 OrbitTargetGoal
- [ ] **追逐阶段**：当与目标距离 > 环绕距离时，以最大速度向目标移动
- [ ] **环绕阶段**：当距离 < 环绕距离时，围绕目标做水平圆周运动
  - 环绕半径由舰种决定（驱逐近、战列远）
  - 当距离 > 1.2×环绕距离时切回追逐
- [ ] **轨迹平滑**：追逐↔环绕切换时速度向量做线性插值（约1秒过渡）
- [ ] **射击触发**：目标进入射程后持续调用攻击Goal

#### 2.3 攻击Goals
- [ ] `CannonAttackGoal`：
  - 冷却间隔由装填速度决定
  - 生成 `DeepOceanProjectileEntity`，使用 `ParabolicCalculator` 计算弹道
  - 每N发混合一发追踪弹（可配置最小/最大间隔）
- [ ] `TorpedoAttackGoal`：
  - 进入雷击距离后发射鱼雷扇形散布
  - 复用现有 `TorpedoEntity`
- [ ] `AircraftLaunchGoal`（轻母/航母专用）：
  - 生成 `AircraftEntity`（敌方），AI与玩家飞机一致但目标为玩家
  - 甲板容量限制、整备间隔
- [ ] `SubmergeGoal`（潜艇专用）：
  - 进入战斗时下潜，获得隐身Buff
  - 水下使用鱼雷攻击
  - 浮出水面时机：弹药耗尽/被深弹命中/滞空时间到

#### 2.4 集群警戒 FleetAlertGoal
- [ ] 每tick检查所属集群状态
- [ ] 集群目标传播（调用 `FleetGroupManager` 接口）
- [ ] 优先级低于直接被攻击

### Phase 3：弹道系统

#### 3.1 DeepOceanProjectileEntity
- [ ] 继承 `ThrowableProjectile`
- [ ] 弹道类型枚举：`PARABOLIC`（抛物线）、`PARABOLIC_TRACKING`（追踪）、`DIRECT`（直射）
- [ ] 碰撞后行为：碰炸（原版爆炸，范围可配置）
- [ ] 友军伤害检查：不伤害其他深海单位

#### 3.2 ParabolicCalculator
- [ ] 输入：发射位置、目标位置、目标速度向量、最小/最大抛高
- [ ] 根据距离线性插值抛高：距离0→最小抛高，最大射程→最大抛高
- [ ] **提前量计算**：预测目标匀速直线运动位置，使炮弹落点=预测位置
- [ ] 返回初速度向量

#### 3.3 TrackingCalculator（追踪弹道）
- [ ] 炮弹过最高点后开始追踪目标
- [ ] **比例制导**（Proportional Navigation）：转弯率 = N × 视线角速率
- [ ] 最大过载限制（使弹道看起来自然，不会90度急转）

#### 3.4 ProximityFuse（近炸引信）
- [ ] 每tick检测周围实体
- [ ] 当距目标 < 引爆距离时触发爆炸
- [ ] 用于防空弹和部分深海炮弹

## 注意事项

### 已踩坑回顾（从CLAUDE.md）
- 实体属性注册必须在 `EntityAttributeCreationEvent` 中（`ModEventBus`）
- 客户端描边用 `isCurrentlyGlowing()` 重写，不要用 `setGlowingTag`
- `setRemainingFireTicks(int ticks)` 而非 `setSecondsOnFire`
- 非LivingEntity发射抛射物：手动 `setPos/setDeltaMovement/setOwner`
- StreamCodec 超6字段用手写 `StreamCodec.of(encoder, decoder)`

### 本版本新增潜在坑
- **Mob AI Goal优先级**：Priority数字越小优先级越高，环绕Goal和攻击Goal需仔细排列
- **水上移动**：Mob默认不会水上行走，需自定义 `MoveControl` 或在 `travel()` 中处理
- **集群跨区块**：集群成员可能分布在不同区块，需确保卸载/加载时集群状态正确恢复
- **深海飞机归属**：深海航母放飞的 `AircraftEntity` 需标记为敌方，玩家飞机的友军判定要排除它们
- **弹道精度**：`ParabolicCalculator` 需考虑tick离散化导致的精度误差，建议用迭代法收敛

### 性能关注点
- 集群 `FleetGroupManager` 不要每tick遍历所有集群，用事件驱动（成员发现目标时通知集群）
- 深海弹道计算缓存结果，同一tick对同一目标不重复计算
- 环绕AI的路径计算频率不宜过高（每10-20tick重算一次方向即可）
- 深海航母飞机数量上限，防止大量飞机实体造成lag

## 验收标准

1. **集群行为正确**：一个深海发现玩家后，同集群所有IDLE成员转入战斗
2. **环绕射击流畅**：深海围绕玩家平滑环绕，不出现抖动或穿墙
3. **弹道表现自然**：抛物线炮弹有明显弧线，追踪弹过最高点后平滑转向目标
4. **潜艇隐身可反制**：玩家装备声呐后能看到下潜的深海潜艇发光轮廓
5. **副本兼容**：新深海实体能在副本 `EnemySetData` 中正确生成和战斗
6. **舰娘NPC可交互**：右键弹出基础对话界面（内容为占位文本）
7. **不崩溃/不卡顿**：50个深海NPC同时战斗时服务端TPS ≥ 18
