# 写给人类程序员

> 本项目由 Claude Code (AI) 协助开发。这份文档面向接手或参与本项目的真实人类，帮你快速理解代码结构、避免踩坑。

---

## 一、项目概述

| 项目 | 值 |
|------|----|
| Mod ID | `piranport` |
| 根包 | `com.piranport` |
| 平台 | NeoForge 21.1.220 / Minecraft 1.21.1 / Java 21 |
| 当前版本 | v0.0.11-alpha（Ruins — 遗迹/深海NPC/集群AI/弹道系统/合成设施/占位物品/调试命令/舰娘NPC框架） |
| 构建工具 | Gradle + ModDevGradle |
| JAR 产物 | `build/libs/piranport-0.0.11.jar` |

## 三、核心架构模式

### 3.1 无 GUI 模式变身机制（舰R模式）

变身状态由**副手槽**驱动（`tickInventoryLoadCheck` 每 tick 检测）：

- 副手放入 ShipCoreItem → 自动 `setTransformed(true)` + `applyAttributes` + `refillFuel` + 消息
- 副手取走 ShipCoreItem → 自动 `setTransformed(false)` + `removeAttributes` + 召回飞机 + 消息
- `ShipCoreItem.use()` 的 Shift+右键分支在无GUI模式直接 `return pass`（禁用）
- `KANSEN_MODE` 配置项（默认 true）强制启用无GUI模式，覆盖 `shipCoreGuiEnabled`

护甲板通过 `overrideOtherStackedOnMe` 以收纳袋方式存入核心（`SHIP_CORE_ARMOR` DataComponent），存入的护甲才计入负重和护甲加成；背包里散放的护甲板不再有效。

无GUI模式专属操作：
- **空手右键**：召回所有舰载机
- **H键**：切换战斗机自动升空
- **U键**：切换战斗机对地攻击模式（默认仅对空）
- **R键**：手动装填（大中口径火炮）

### 3.2 注册流程（必须理解）

DataComponents → Blocks → Items → CreativeTabs → MenuTypes → EntityTypes → MobEffects → BlockEntityTypes → RecipeTypes

KeyMapping 在 `ClientEvents.registerKeyMappings()` 注册（MOD 总线，客户端）。

### 3.3 功能方块标准四件套

每个带 GUI 的功能方块：`XxxBlock` + `XxxBlockEntity` + `XxxMenu` + `XxxScreen`，加上对应注册和 `ClientEvents` 绑定。

### 3.4 AircraftEntity 状态机（6种状态）

```
LAUNCHING(30tick) → CRUISING(绕轨) → ATTACKING → RETURNING → REMOVED
                        ↓
                  RECON_ACTIVE（侦察机专属）
```

- 继承 `Entity`，不继承 Mob，`noPhysics = true`，手动管理移动
- 攻击分派在 `tickAttacking()` 中按 `aircraftType` switch
- FIGHTER 特有 `resolveFighterTarget()` — 优先火控目标，其次敌方飞机，最后 Monster
- 攻击机（轰炸机+鱼雷机）不攻击空中目标，仅战斗机参与对空作战
- 四重防御：滞空上限 / 距离TP / 卡死检测 / 右键召回
- 返航物品通过 `coreInventorySlot` 精准写回，返回时 `currentFuel` 清零
- 每种飞机有血量（FIGHTER=20，RECON=10 等），被击落无物品返还
- 击坠掉落物使用 `AircraftDropEntity`（防爆+浮水，类似下界之星）
- **转向半径约束**：`constrainTurnRadius()` 限制每 tick 水平偏转角，公式 ω=v/r（`TURN_RADIUS=2.0` 格），飞机画 2 格半径圆弧转弯，不可瞬转。`applyTurnRadiusConstraint()` 同时强制最低 5% 面板速度防止悬停
- **水平轰炸机轰炸航线模式**：`tickLevelBomberAttack()` 实现爬升→锁定航向→直线飞越投弹→飞过 10 格→重新定位循环。投弹窗口为与目标水平距离 ≤3 格时触发

### 3.5 燃油系统

舰装核心有 `SHIP_CORE_FUEL`（`FuelData`）组件，存储 currentFuel/maxFuel。

- 支持燃料类型：航空燃料（FUEL）、煤炭、烈焰棒
- 飞机飞行中每 4 tick 消耗 1 燃油
- 燃油耗尽 → 直接返航（`isForcedReturn`）
- 侦察机无输入时沿当前朝向以最低速度前进（禁止悬停）

### 3.6 FlightGroupData 结构

存在**核心物品的 DataComponent** 上，不在飞机物品上：

```java
groups: [ { slotIndices: Set<Integer>, slotAmmoTypes: Map<Integer,String>,
            attackMode: FOCUS|SPREAD|FOLLOW, sortOrder: int } × 4 ]
```

弹种是**每架飞机独立配置**（`slotAmmoTypes`），不是整组共享。
FOLLOW 模式：巡航时跟随玩家的侦察机轨道飞行；有锁定目标时进入 ATTACKING。

### 3.7 火控系统数据流

```
客户端 P/O/I 按键
  → FireControlPayload(C2S)
  → 服务端 FireControlManager.lock/addTarget/clearTargets(ownerUUID, targetUUID)
  → FireControlSyncPayload(S2C)
  → 客户端 ClientFireControlData（只读，供 HUD 渲染）
  → AircraftEntity.resolveTarget() 直接读 FireControlManager（服务端）
```

火控描边：客户端记分板队伍 `pp_fc_target`（RED色），`ClientTickHandler.syncFcTeam()` 每tick同步。

自动升空时 `GameEvents.tickAutoLaunchFighters()` 在服务端直接调用 `FireControlManager.lock()` 锁定幻翼，无需经过 P 键流程。

战斗机默认**仅对空**（`isFighterAirOnly=true`），U 键切换对地攻击模式。

### 3.8 编组GUI无槽模式

`FlightGroupMenu` 继承 `AbstractContainerMenu`，但**0个容器槽**。玩家物品栏槽位设 x=-2000 隐藏。所有交互通过 C2S Payload（`FlightGroupUpdatePayload`）直接修改核心 DataComponent，不走 slot 机制。

### 3.9 加工站 IItemHandler Capability

厨锅、砧板、石磨均通过 `RegisterCapabilitiesEvent` 注册 `Capabilities.ItemHandler.BLOCK`：

- **厨锅**：`direction == DOWN` → 输出接口（只读输出槽）；其他方向 → 输入接口（仅武器槽）
- **砧板**：单槽，任意方向均可读写
- **石磨**：4输入2输出，任意方向均可（AE2 已兼容）

发射器联动：`CuttingBoardBlock.neighborChanged()` 检测相邻方块是否为 `DispenserBlock` 且 `TRIGGERED=true` 时自动执行一次切割。

### 3.10 侦察机系统

```
玩家 Shift+右键放飞侦察机
  → launchAircraft → AircraftEntity（RECON 类型）
  → LAUNCHING → RECON_ACTIVE（跳过 CRUISING）
  → ReconManager.startRecon(playerUUID, reconUUID)
  → ReconStartPayload(S2C) → 客户端 Minecraft.setCameraEntity(reconEntity)
  → 客户端每 tick: ReconControlPayload(dx,dy,dz)(C2S)
  → 服务端 ReconManager.handleControl() → AircraftEntity.tickReconActive()
  → 维护周围 3×3 区块强制加载
```

侦察机模式下禁用空手右键操作（防止皮肤还原和舰载机召回出错）。

### 3.11 战斗机自动升空

舰装配置开关存在舰装核心的 `SHIP_AUTO_LAUNCH`（Boolean DataComponent）上。GUI模式通过 ShipCoreScreen 按钮切换；无GUI模式通过 **H键** 切换。

```
每 40 tick（服务端 onPlayerTick）
  → 检查 SHIP_AUTO_LAUNCH == true
  → 检测 64 格内敌对飞行生物（Enemy + FlyingMob/Phantom/Vex，含幻翼）
  → 若无活跃火控锁定：FireControlManager.lock(nearest)
  → ShipCoreItem.tryAutoLaunchFighter()
      → refillAircraftFuel()（从弹药库取航空燃料补满飞机）
      → 找第一个有弹药配置的 FIGHTER 武器槽
      → launchAircraft()（消耗弹药、清空槽位、生成实体）
  → 检测 32 格内空中 Enemy（排除水下/地面）
  → ShipCoreItem.tryAutoFireAntiAirMissile() 自动发射防空导弹

注意：Phantom 继承自 FlyingMob 而非 Monster，所有相关敌对筛选必须用 Enemy 接口，
否则 H 模式会漏识别幻翼，手动发射的防空导弹也无法自动追踪幻翼。
```

### 3.12 友军伤害防护

`FriendlyFireHelper.shouldBlockHit()` 被所有抛射物共用：
- `FRIENDLY_FIRE_ENABLED=false` 时跳过对其他玩家的伤害
- 始终跳过对自己飞机的伤害
- 抛射物记录 owner UUID，击落飞机时通过 owner 追溯到发射者

### 3.13 手动装填（R键）

大中口径火炮支持 R 键手动装填（`ManualReloadPayload` C2S）。弹药选择逻辑与自动装弹一致，从背包中消耗对应口径炮弹。

### 3.14 三式弹系统

三式弹（Sanshiki）为特殊炮弹类型，发射后在空中散布为 64 发 `SanshikiPelletEntity`（8×8锥形散布），每发子弹丸伤害为同口径 HE 弹的 1/4。适合对空射击。

### 3.15 皮肤系统

- `SkinCoreItem`：右键激活/切换皮肤（skinId 1~3）
- `SkinManager`（服务端）：维护 playerUUID → skinId 映射
- `ClientSkinData`：客户端皮肤状态
- `SkinOverlayLayer`：渲染层，将 `assets/piranport/textures/skin/skin_{id}.png`（64×64）叠加到玩家模型
- `SkinSyncPayload` / `SkinRevertPayload`：S2C 同步

### 3.16 副本系统

独立维度 `piranport:dungeon`，JSON驱动关卡数据，实例化战斗场，大厅组队机制。

- **数据层**：`data/piranport/dungeon/` 下 chapters/stages/enemy_sets JSON，`DungeonDataLoader` 加载
- **实例管理**：`DungeonInstanceManager` 管理多个 `DungeonInstance`，每个包含 `NodeBattleField`
- **脚本化节点**：`NodeData.script` 字段触发脚本分发；`DungeonScriptManager` 每tick驱动
- **运输机**：`DungeonTransportPlaneEntity` 直线飞行+空投+爬升消失
- **低阶驱逐舰**：`LowTierDestroyerEntity`（Monster），水面轨道巡航，30格侦测，100tick开炮
- **JEI兼容**：`PiranPortJEIPlugin` 注册3个加工站配方Category，`localRuntime` 软依赖

### 3.17 深海NPC系统（v0.0.11）

9种深海敌人继承自 `AbstractDeepOceanEntity`（extends PathfinderMob），通过虚方法覆盖实现差异化：

```
AbstractDeepOceanEntity
├── getOrbitDistance()       → 环绕半径
├── getFireInterval()       → 炮击间隔（tick）
├── getShellDamage()        → 炮弹伤害
├── getExplosionPower()     → 爆炸力度
├── getTrackingIntervalMin/Max() → 每N发混入1发追踪弹
├── canUseTorpedoes()       → 是否可发射鱼雷
├── canLaunchAircraft()     → 是否可放飞飞机
├── getTorpedoDamage()      → 鱼雷伤害
└── getMaxAircraft()        → 最大同时飞机数
```

**基类核心行为**：
- 水面漂浮：tick() 中检测水位，保持眼睛露出水面
- 沉没动画：死亡时 40 tick 逐帧下沉 + 气泡粒子，然后 `remove(KILLED)`
- 舰队状态：`DATA_FLEET_STATE` 同步数据（IDLE/ALERT/COMBAT），影响渲染着色
- 溅水粒子：每 10 tick 生成船尾迹象

**AI Goal 优先级（以驱逐舰为例）**：
1. `OrbitTargetGoal` — CHASE/ORBIT/RETREAT 三阶段移动，20tick 平滑过渡
2. `CannonAttackGoal` — 每 N tick 发射 `DeepOceanProjectileEntity`，混入追踪弹
3. `TorpedoAttackGoal` — 20格内发射3条扇形鱼雷（8度间距），200tick冷却
4. `FleetAlertGoal` — 20格内同 `fleetGroupId` 的实体共享目标
5. `IdleWanderGoal` — 无目标时漫游

**航母特有**：`AircraftLaunchGoal`（120-200tick检测一次，轻航最多2架，航母最多4架）
**潜艇特有**：`SubmergeGoal`（水下隐身，被声呐GLOWING效果发现后失去隐身）

**数据驱动**：`data/piranport/npc/deep_ocean/*.json`，`DeepOceanDataLoader` 加载，支持热配置敌人属性。

### 3.18 弹道系统

`DeepOceanProjectileEntity` 支持三种弹道模式：

| 模式 | 说明 |
|------|------|
| `PARABOLIC` | 标准抛物线，根据距离线性插值抛高 |
| `PARABOLIC_TRACKING` | 上升时抛物线，过最高点后启用比例导航制导 |
| `DIRECT` | 直线无重力（电磁炮模式） |

弹道计算工具类（`npc/ai/ballistic/`）：
- `ParabolicCalculator` — 根据距离计算发射仰角 + 提前量补偿
- `TrackingCalculator` — 比例导航制导，`hasPassedApex()` 判断何时启用追踪，最大过载限制转弯率
- `ProximityFuse` — 近炸引信（3.0格范围检测）

弹丸生命周期：200 tick（10秒）后自动销毁，爆炸受 `EXPLOSION_BLOCK_DAMAGE` config 控制。

### 3.19 导弹与深弹系统

**导弹**（`MissileEntity`）：
- 三种类型：`ANTI_SHIP`（反舰）/ `ANTI_AIR`（防空）/ `ROCKET`（火箭）
- 制导追踪：发射后锁定目标，持续修正飞行方向
- 防空导弹自动升空模式（H键）：自动搜索并打击空中飞行目标（所有 `Enemy` 接口的敌对生物，含幻翼；排除水下/地面目标和容器实体）
- 反舰/火箭导弹为手动装填模式（`isManualReload()`），防空导弹为自动模式（`cooldownTicks` 控制间隔）
- 目标锁定排除规则：自动瞄准限定 `Enemy`（Monster/Phantom/Vex/Ghast 等）+ 火控排除 `Container` 类实体

**深弹**（`DepthChargeEntity`）：
- `DepthChargeLauncherItem` 三型号
- 近炸引信 + 区域魔法伤害
- 独立 `DEPTH_CHARGE` 武器分类（`WeaponCategory`）

**命名鱼雷**：15种（G7a/G7e/MK系列/九式/零式），各有独立伤害/航程/速度属性，`TorpedoEntity` 读取装填物品数据。

### 3.20 合成设施系统

| `AmmoWorkbenchBlock` | 合成弹药 | 1输出槽，背包联动 |
| `WeaponWorkbenchBlock` | 合成武器 | 8槽：蓝图1 + 原料6 + 产物1 |

**武器合成台特性**：

### 3.21 遗迹与世界生成（v0.0.11）

四类主世界遗迹（Jigsaw/SinglePool 结构）：

| 遗迹类型 | 规模 | 内容 |
|----------|------|------|
| `portal_ruin` | 小型 | 传送门框架 + 战利品箱 |
| `supply_depot` | 中型 | 补给物资 + 战利品箱 |
| `outpost` | 大型 | 驻守深海NPC + 战利品 |
| `abyssal_base` | 超大型 | 大量深海NPC + 高级战利品 |

**结构处理器链**：
1. `LootChestProcessor` — 检测箱子方块，注入 `LootTable` + 随机种子
2. `RuinDegradationProcessor` — 模拟战场残骸（70%→空气，30%→圆石；不损坏箱子/spawner）
3. `AbyssalSpawnerBlock` — 结构首次加载时生成深海NPC后自毁

**一次性Spawner机制**：`AbyssalSpawnerBlockEntity` 存储 entityType/count/clusterUUID/patrolRadius，首次 tick 时生成NPC并加入同一 `FleetGroup`，然后自毁。

### 3.25 火炮联装数系统

- 单装（1）、双联（2）、三联（3）等

- 右键发射时消耗已装填的全部炮管弹药
- Tooltip 在 Shift 展开时显示联装数

### 3.26 武器空耐久条提示

`WeaponReloadDecorator.render()` 在手动装填类武器未装填时显示黑色空耐久条，提醒玩家需要装弹：

- 适用于：大中口径火炮（非小口径）、鱼雷发射器、手动装填导弹发射器（反舰SY1/鱼叉/火箭）
- 触发条件：`LoadedAmmo` 为空 + 自动装填模式关闭
- 与冷却条（红→黄→绿）共用同一 Decorator，互斥显示

### 3.27 创造模式标签页重整

新增 **"皮兰港设施"** 创造标签页（`FACILITY_TAB`），将功能性方块和工具从"强化"标签迁出：

### 3.28 占位物品

- **国旗×7**：日/英/美/德/法/意/中（皮肤合成材料）
- **无序意志碎片×9**（α~ι）：传送门激活材料

## 四、已踩坑清单（必读）

| 坑 | 错误写法 | 正确写法 |
|----|---------|---------|
| 酿造事件总线 | `modEventBus.addListener(RegisterBrewingRecipesEvent)` | `NeoForge.EVENT_BUS.addListener(...)` |
| 酿造旧API | `BrewingRecipeRegistry.addRecipe(recipe)` | `event.getBuilder().addRecipe(recipe)` |
| 食物饱和度 | `food.saturationModifier()` | `food.saturation()` |
| 熔炉热源检测 | `AbstractFurnaceBlockEntity.isLit()`（private）| `bs.getBlock() instanceof AbstractFurnaceBlock && bs.getValue(LIT)` |
| PlaceableFoodBlock codec | 单父类 `simpleCodec(Base::new)` | 三个静态内部类各自实现 `simpleCodec` |
| StreamCodec 超6字段 | `StreamCodec.composite(...)` | `StreamCodec.of(encoder, decoder)` 手写 |
| 编组GUI开启 | 客户端 Screen 直接 openMenu | 发 C2S 包 → 服务端 `serverPlayer.openMenu()` |
| 实体着火 | `entity.setSecondsOnFire(int)` | `entity.setRemainingFireTicks(int ticks)` |
| 从非LivingEntity发射抛射物 | 直接 new TorpedoEntity(level, shooter) | `new TorpedoEntity(type, level)` + 手动 `setPos/setDeltaMovement/setOwner` |
| 返回物品到背包 | `player.getMainHandItem()` | 存储 `coreInventorySlot`，返航时 `player.getInventory().getItem(slot)` |
| instanceof 模式变量 | `e instanceof Foo && e.getFoo()` | `e instanceof Foo f && f.getFoo()` |
| Inventory offhand槽位 | 无常量 | `player.getInventory().getItem(40)` |
| Phantom 不是 Monster | `instanceof Monster` 无法匹配 Phantom | `instanceof Phantom` 或走 FireControlManager 锁定路径 |
| 武器冷却共享 | `player.getCooldowns().addCooldown(this, ...)` | 用 `SLOT_COOLDOWNS` DataComponent 分槽记录 |
| 客户端实体描边 | `setGlowingTag(true)` 客户端无效 | 重写 `isCurrentlyGlowing()`，客户端读静态标志 |
| 彩色实体描边 | `entity.setGlowingTag` + team color 不同步 | 客户端记分板队伍 + `ChatFormatting.RED` 等颜色，每tick `syncFcTeam()` |
| 自定义维度注册 | 代码注册 dimension | `data/piranport/dimension/` + `dimension_type/` JSON 文件 |
| Patchouli 书籍结构 | `data/` 下 → `use_resource_pack` 异常 | `book.json` 加 `"use_resource_pack": true`；JSON 放 `assets/` |
| Patchouli 软依赖 | 直接Maven成硬依赖 | `localRuntime` + BlameJared maven + 反射调用 |
| 条件性BiomeModifier | 标准 `add_features` 无法动态开关 | 自定义 `BiomeModifier` + config 读取 |
| 导弹发射器构造NPE | `getDefaultItem()` 在 `super()` 期间 displayItemId 为 null | 延迟初始化或 null 检查 |
| 从非LivingEntity发射弹丸 | 构造函数会把位置设到 shooter | `new Entity(type, level)` 后手动 `setPos/setDeltaMovement/setOwner` |
| 导弹自动锁定炸鱼/炸箱子 | `instanceof Monster` 匹配水下/容器实体 | 排除水下目标（`isUnderWater()`）+ 排除容器实体（`instanceof Container`） |
| 防空导弹锁定非飞行目标 | `isManualReload` 判断错误 | 防空导弹 `MissileType.ANTI_AIR` 走自动模式，仅锁定 `FlyingMob`/`AircraftEntity` |
| 自动装填导弹的LoadedAmmo检查 | 自动发射导弹也显示LoadedAmmo tooltip | `isManualReload()` 区分，自动装填导弹不显示弹药tooltip |
| 防空导弹/H模式不识别幻翼 | `Phantom extends FlyingMob`，不是 `Monster` | 所有敌对筛选用 `instanceof Enemy` 接口，覆盖 Monster/Phantom/Vex/Ghast 等 |

## 六、如何添加新内容

### 添加普通物品

1. `ModItems.java`：`ITEMS.registerSimpleItem("id")`
2. `models/item/id.json`：`item/generated` 父模型
3. `lang/*.json`：添加翻译（`zh_cn.json` + `en_us.json`）
4. `ModCreativeTabs.java`：加到创造模式标签

### 添加食物（带放置功能）

```java
ITEMS.register("my_food", () -> new ModFoodItem(new Item.Properties()
    .food(fp(5, 6.3f)
        .effect(new MobEffectInstance(MobEffects.MOVEMENT_SPEED, 200, 1), 1.0f)
        .build())
    .component(ModDataComponents.PLACEABLE_INFO.get(), new PlaceableInfo("plate", 2))));
```

### 添加厨锅/石磨/砧板配方

在 `data/piranport/recipe/` 新建 JSON，`type` 字段分别为：
- `piranport:cooking_pot`（含 `cookingTime` 字段）
- `piranport:stone_mill`
- `piranport:cutting_board`（含 `cuts` 字段）

### 添加酿造配方

在 `ModBrewingRecipes.java` 的 `register()` 方法中追加 `builder.addRecipe()`。

### 添加新的舰装配置开关

1. `ModDataComponents.java`：注册新的 Boolean DataComponent（参考 `SHIP_AUTO_LAUNCH`）
2. 新建对应 Payload（参考 `AutoLaunchTogglePayload`）
3. `ModPackets.java`：注册新 Payload
4. `ShipCoreMenu.java`：新增 `isXxx()` getter
5. `ShipCoreScreen.java`：在"舰装配置"区域增加新按钮
6. 在 `GameEvents.java` 或专门的服务端事件类实现服务端逻辑

### 添加新的副本关卡

1. `data/piranport/dungeon/chapters/` 下新建 chapter JSON
2. `data/piranport/dungeon/stages/` 下新建 stage JSON（引用 chapter）
3. `data/piranport/dungeon/enemy_sets/` 下新建 enemy_set JSON
4. 若需脚本化节点：实现 `DungeonScript` 接口，在 `DungeonScriptManager` 注册

### 添加新的深海NPC

1. `npc/deepocean/` 下新建类，继承 `AbstractDeepOceanEntity`
2. 覆盖虚方法配置属性（`getOrbitDistance`/`getFireInterval`/`getShellDamage` 等）
3. `registerGoals()` 中按优先级注册 AI Goal（参考已有子类）
4. `CommonEvents.java` 中 `RegisterSpawnPlacementsEvent` 注册属性
5. `ModEntityTypes.java` 注册实体类型
6. `ModItems.java` 注册刷怪蛋
7. `data/piranport/npc/deep_ocean/` 下新建 JSON 配置
8. `ClientEvents.java` 中注册渲染器（使用 `DeepOceanRenderer`）
9. `lang/*.json` 添加翻译

### 添加新的弹药/武器合成配方

**弹药配方**：在 `AmmoRecipeRegistry` 中 `registerAll()` 添加 `register()` 调用。

**武器配方**：在 `WeaponWorkbenchRecipeRegistry` 中添加配方注册。配方需指定标签页、蓝图、所需材料列表和产物。

### 添加新的遗迹类型

1. 游戏内搭建结构模板，用结构方块导出为 `.nbt` 放入 `data/piranport/structure/`
2. `data/piranport/worldgen/structure/` 下新建结构 JSON
3. `data/piranport/worldgen/structure_set/` 下配置间距（spacing/separation）
4. `ModStructures.java` 注册结构常量
5. 如需战利品：在 `data/piranport/loot_table/` 下新建战利品表
6. 如需敌人：在结构中放置 `AbyssalSpawnerBlock`（设置 entityType/count/clusterUUID）
