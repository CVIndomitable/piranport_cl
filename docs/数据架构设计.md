# 皮兰港 — 数据架构设计

## 概述

皮兰港的游戏数据（伤害、冷却、负重等）目前采用**混合存储方式**，没有统一的数据驱动框架。本文档梳理各类数据的存储位置和访问方式。

---

## 1. 舰炮系统

### 存储位置
`src/main/java/com/piranport/item/ShipCoreItem.java` — 私有静态方法 (行 825-858)

### 访问方式
发射炮弹时从 `ShipCoreItem` 调用 `getGunXxx()` 系列方法，根据物品 ID 匹配返回数值：

```java
private static float getGunDamage(ItemStack weapon) {
    if (weapon.is(ModItems.SMALL_GUN.get())) return 6f;
    if (weapon.is(ModItems.MEDIUM_GUN.get())) return 12f;
    if (weapon.is(ModItems.LARGE_GUN.get())) return 20f;
    return 6f;
}

private static int getGunCooldown(ItemStack weapon) {
    if (weapon.is(ModItems.SMALL_GUN.get())) return 30;
    if (weapon.is(ModItems.MEDIUM_GUN.get())) return 50;
    if (weapon.is(ModItems.LARGE_GUN.get())) return 80;
    return 30;
}
// 同理: getExplosionPower, getProjectileVelocity, getProjectileInaccuracy
```

### 数值表

| 属性 | 小型炮 | 中型炮 | 大型炮 |
|------|--------|--------|--------|
| **伤害** (damage) | 6f | 12f | 20f |
| **冷却** (cooldown) | 30 tick | 50 tick | 80 tick |
| **爆炸力** (explosionPower) | 1.0f | 1.5f | 2.0f |
| **发射速度** (velocity) | 2.0f | 2.5f | 3.0f |
| **精准度** (inaccuracy) | 1.5f | 1.0f | 0.5f |

### 炮弹特效

#### HE弹（高爆）
- 在 `CannonProjectileEntity:57` 触发爆炸效果
- 爆炸半径由 `explosionPower` 决定

#### AP弹（穿甲）
- 在 `CannonProjectileEntity:61` 增加 **1.3倍伤害倍率**
- 无视目标 **50%护甲**（临时减少护甲属性）
- 代码：
```java
float apDamage = damage * 1.3f;
double halfArmor = living.getAttributeValue(Attributes.ARMOR) * 0.5;
armorAttr.addTransientModifier(new AttributeModifier(..., -halfArmor, ...));
```

### 发射流程
1. 玩家右键（持有舰装核心）
2. `ShipCoreItem.fireCannonAutoMode()` 或 `fireCannonManualMode()` 被调用
3. 调用 `getGunDamage(weapon)` 等方法获取参数
4. 创建 `CannonProjectileEntity` 实例，传入数值
5. 炮弹实体运行时使用这些数值进行伤害计算

## 2. 飞机系统

### 存储位置

在物品注册时添加：`src/main/java/com/piranport/registry/ModItems.java` (行 278-316)

### 定义

```java
public record AircraftInfo(
    AircraftType aircraftType,      // FIGHTER / DIVE_BOMBER / TORPEDO_BOMBER / LEVEL_BOMBER / RECON
    int fuelCapacity,               // 燃料容量
    int ammoCapacity,               // 弹药容量
    int currentFuel,                // 当前燃料（发射时会更新）
```

### 数值表

| 飞机类型 | 燃料容量 | 弹药容量 | 面板伤害 | 航速 | 负重 | 轰炸方式 |
|---------|---------|---------|---------|------|------|---------|
| 战斗机 (FIGHTER) | 1200 | 64 | 18f | 1.8f | 12 | DIVE |
| 俯冲轰炸机 (DIVE_BOMBER) | 1200 | 8 | 24f | 1.4f | 16 | DIVE |
| 鱼雷轰炸机 (TORPEDO_BOMBER) | 1200 | 6 | 30f | 1.2f | 20 | DIVE |

### 访问方式


### 攻击效果

| 飞机类型 | 攻击方式 | 伤害计算 | 附加效果 |
|---------|---------|---------|---------|
| FIGHTER | 近战（dist < 2.5） | `panelDamage` × 编队系数 | — |
| DIVE_BOMBER | 俯冲接触 | `panelDamage × 1.5` | 50% 概率着火 30s |
| TORPEDO_BOMBER | 投鱼雷（dist 10-16） | 复用鱼雷伤害 | 进水 debuff |
| LEVEL_BOMBER | 投航弹（正上方） | `panelDamage × 1.5` | 爆炸伤害 |
| RECON | — | 0（无攻击力） | — |

```java
private static Map<Item, Integer> weaponLoadMap;

    if (weaponLoadMap == null) {
        weaponLoadMap = new IdentityHashMap<>();
        weaponLoadMap.put(ModItems.SMALL_GUN.get(), 6);
        weaponLoadMap.put(ModItems.MEDIUM_GUN.get(), 16);
```

### 计算方式

```java
public static int getItemLoad(ItemStack stack) {
    Integer load = getWeaponLoadMap().get(stack.getItem());
    if (load != null) return load;
    if (stack.getItem() instanceof ArmorPlateItem plate) {
        return plate.getWeight();
    }
    if (stack.getItem() instanceof AircraftItem) {
        AircraftInfo info = stack.get(ModDataComponents.AIRCRAFT_INFO.get());
        return info != null ? info.weight() : 0;
    }
    return 0;
}
```

### 舰装容量

## 5. 舰装属性系统

### 存储位置
`src/main/java/com/piranport/combat/TransformationManager.java` — 动态计算

### 属性加成

#### 舰装栏（GUI模式）
| 属性 | 来源 |
|------|------|
| 生命值 | ShipType.maxHealth（固定） |
| 护甲 | 装备的装甲板总和 |
| 移速 | 负重比率（1 - load/capacity） |
| 攻击伤害 | 所有火炮求和（需代码补全） |

#### 无GUI模式（副手驱动）
核心护甲存储在 `SHIP_CORE_ARMOR` DataComponent（ItemContainerContents）：
```java
public static int getCoreArmorBonus(ItemStack core) {
    ItemContainerContents contents = core.get(ModDataComponents.SHIP_CORE_ARMOR.get());
    if (contents == null) return 0;
    int bonus = 0;
    for (ItemStack armor : contents.getItems()) {
        if (armor.getItem() instanceof ArmorPlateItem plate) {
            bonus += plate.getArmorBonus();
        }
    }
    return bonus;
}
```

### 减速计算

```java
float loadRatio = (float) load / capacity;
float speedMultiplier = Math.max(0.1f, 1.0f - loadRatio);
// 例：75/100 = 0.75 load → 25% 移速减速
```

## 6. 特殊Buff系统

### 存储位置
`src/main/java/com/piranport/registry/ModMobEffects.java` + `ModItems.java` 食物定义

### 自定义Buff

| Buff | 效果 | 来源 | 代码位置 |
|------|------|------|---------|
| **RELOAD_BOOST** | 装填加速（减冷却） | 龙田烧、鱼雷果汁 | `ReloadBoostEffect` + `TransformationManager.boostedCooldown()` |
| **EVASION** | 高速规避（减伤） | 天妇罗、格瓦斯 | `EvasionHandler.onEntityHurt()` |
| **FLAMMABLE** | 易燃易爆（增伤） | 变身 + 有燃料飞机 | `FlammableEffect` |
| **FLOODING** | 进水 debuff | 鱼雷命中 | `FloodingEffect` |

### 装填加速计算

```java
public static int boostedCooldown(Player player, int baseCooldown) {
    if (player == null) return baseCooldown;
    MobEffectInstance reloadBoost = player.getEffect(ModMobEffects.RELOAD_BOOST.get());
    if (reloadBoost == null) return baseCooldown;
    // Amplifier 0 = 80%，Amplifier 1 = 60%
    float multiplier = 1.0f - (reloadBoost.getAmplifier() + 1) * 0.2f;
    return Math.round(baseCooldown * multiplier);
}
```

### 高速规避伤害减免

```java
public class EvasionHandler {
    public static float applyEvasion(Player player, float damage, DamageSource source) {
        if (shouldIgnoreEvasion(source)) return damage;  // 排除 BYPASSES_INVULNERABILITY 等
        MobEffectInstance evasion = player.getEffect(ModMobEffects.EVASION.get());
        if (evasion == null) return damage;
        // Amplifier 0 = 20% 减伤，Amplifier 1 = 40% 减伤
        float reduction = (evasion.getAmplifier() + 1) * 0.2f;
        return damage * (1.0f - reduction);
    }
}
```

## 8. 问题与改进方向

### 现状问题

1. **舰炮数据硬编码** — `ShipCoreItem` 的 if-else 链，添加新炮需改代码
2. **存储方式不统一** — 舰炮用方法，飞机用 DataComponent，负重用 HashMap
3. **缺少中央注册表** — 各子系统自行维护数据，容易出现不一致

### 改进方向

#### 方案A：统一为 DataComponent（推荐）
```java
public record CannonSpec(
    float damage,
    int cooldown,
    float explosionPower,
    float velocity,
    float inaccuracy
) { }

// ModItems.java 注册时
new CannonItem(new Item.Properties()
    .component(ModDataComponents.CANNON_SPEC.get(),
        new CannonSpec(6f, 30, 1.0f, 2.0f, 1.5f)))
```

**优点**：
- 与飞机系统一致
- 物品可自定义数值
- 便于未来扩展

#### 方案B：统一为中央查表（次选）
```java
// GunStats.java
public class GunStats {
    public static final Map<ResourceLocation, CannonStats> REGISTRY = new HashMap<>();

    static {
        REGISTRY.put(ModItems.SMALL_GUN_ID, new CannonStats(...));
    }
}
```

**优点**：
- 所有数据在一个文件
- 易于批量编辑
- 便于工具生成

#### 方案C：数据驱动 JSON（最灵活）
```json
{
  "piranport:small_gun": {
    "damage": 6,
    "cooldown": 30,
    "explosionPower": 1.0,
    "velocity": 2.0,
    "inaccuracy": 1.5
  }
}
```

**优点**：
- 无需重编译即可调整
- 易于平衡师调参
- 便于模组整合

### 数据修改检查清单

修改任何数值时，检查以下关联数据：
- [ ] 负重是否与 `TransformationManager` 一致
- [ ] 冷却是否与 HUD 显示时间一致
- [ ] 伤害是否与 Buff 倍率相匹配
- [ ] 舰装容量是否足以放置所有装备
- [ ] 配置项是否能影响此数值
