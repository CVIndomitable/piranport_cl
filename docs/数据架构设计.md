# 皮兰港 — 数据架构设计

## 概述

> 当前覆盖版本：v0.0.11 Ruins（2026-04-18）。版本主题速查见 [`version_roadmap.md`](version_roadmap.md)。

皮兰港的游戏数据（伤害、冷却、负重等）采用**混合存储方式**：
- **构造参数**：舰炮、鱼雷发射器、导弹发射器、深弹发射器、装甲板、引擎
- **DataComponent**：飞机属性、编组数据、冷却数据、燃油数据
- **硬编码常量**：电磁炮、冈格尼尔、特殊道具、舰娘NPC基础属性
- **JSON 数据驱动**：深海NPC属性、副本关卡数据、遗迹战利品表
- **静态注册表**：弹药配方、武器合成配方
- **结构 JSON**：遗迹结构模板 (`data/piranport/structure/`) + `ModStructures` 常量

## 1. 舰炮系统

### 存储位置
- `ShipCoreItem.java` — 发射逻辑 + `getGunXxx()` 系列方法
- `CannonItem.java` — 伤害/冷却/联装数通过 `getDamage()` / `getCooldownTicks()` / `getBarrelCount()` 委托
- `ModItems.java` — 注册时传入数值

### 数值表

| 属性 | 单装小型炮 | 小型炮(联装) | 中型炮 | 大型炮 |
|------|-----------|-------------|--------|--------|
| **伤害** (damage) | 6f | 6f | 12f | 20f |
| **冷却** (cooldown) | 30 tick | 30 tick | 50 tick | 80 tick |
| **联装数** (barrelCount) | 1 | 2 | 1 | 3 |
| **爆炸力** (explosionPower) | 1.0f | 1.0f | 1.5f | 2.0f |
| **发射速度** (velocity) | 2.0f | 2.0f | 2.5f | 3.0f |
| **精准度** (inaccuracy) | 1.5f | 1.5f | 1.0f | 0.5f |

> **联装数说明**：每次射击消耗 `barrelCount` 发弹药，同时发射 `barrelCount` 颗弹丸。装填时也需要 `barrelCount` 发弹药。

### 弹种

#### HE弹（高爆）
- `CannonProjectileEntity` 触发爆炸效果，爆炸半径由 `explosionPower` 决定

#### AP弹（穿甲）
- **1.3倍伤害倍率**，无视目标 **50%护甲**（临时减少护甲属性）

#### VT弹（近炸引信）
- 检测前方 30° 半锥角、5.0 格范围内的实体
- 接近 3.0 格时自动引爆
- 发射后 5 tick 解锁安全保险
- 仅小口径注册：`SMALL_VT_SHELL`

#### 三式弹（散射）
- 发射 64 颗散射弹丸（向日葵/黄金角分布）
- 每颗伤害 = 同口径 HE 基伤 × 1/4
- 三种口径：`SMALL_TYPE3_SHELL`、`MEDIUM_TYPE3_SHELL`、`LARGE_TYPE3_SHELL`

## 2. 飞机系统

### 存储位置
`AircraftInfo` — **DataComponent**（`ModItems.java` 注册时附加）

### 定义

```java
public record AircraftInfo(
    AircraftType aircraftType,      // FIGHTER / DIVE_BOMBER / TORPEDO_BOMBER / LEVEL_BOMBER / ASW / RECON
    int fuelCapacity,               // 燃料容量
    int ammoCapacity,               // 弹药容量
    int currentFuel,                // 当前燃料
    float panelDamage,              // 面板伤害
    float panelSpeed,               // 面板航速
    int weight,                     // 负重
    BombingMode bombingMode         // 轰炸方式
) { }
```

### 数值表

| 飞机类型 | 燃料容量 | 弹药容量 | 面板伤害 | 航速 | 负重 | 轰炸方式 |
|---------|---------|---------|---------|------|------|---------|
| 战斗机 (FIGHTER) | 1200 | 64 | 18f | 1.8f | 12 | DIVE |
| 俯冲轰炸机 (DIVE_BOMBER) | 1200 | 8 | 24f | 1.4f | 16 | DIVE |
| 鱼雷轰炸机 (TORPEDO_BOMBER) | 1200 | 6 | 30f | 1.2f | 20 | DIVE |
| 水平轰炸机 (LEVEL_BOMBER) | 1200 | 8 | 36f | 1.0f | 24 | LEVEL |
| 侦察机 (RECON) | 1500 | 0 | 0f | 1.5f | 8 | DIVE |

### 飞行常量

| 常量 | 值 | 说明 |
|------|-----|------|
| TURN_RADIUS | 2.0 格 | 水平转向弧线半径 |
| FUEL_BURN_INTERVAL | 4 tick | 每4 tick消耗1燃料 |
| 最低前进速度 | panelSpeed × 0.05 | 防悬停（LAUNCHING阶段除外） |
| 最大偏转角/tick | desiredHorizSpeed / TURN_RADIUS (弧度) | 转向角速度限制 |

### 攻击效果

| 飞机类型 | 攻击方式 | 伤害计算 | 附加效果 |
|---------|---------|---------|---------|
| FIGHTER | 近战（dist < 2.5） | `panelDamage` × 编队系数 | — |
| DIVE_BOMBER | 俯冲接触 | `panelDamage × 1.5` | 50% 概率着火 30s |
| TORPEDO_BOMBER | 投鱼雷（dist 10-16） | 复用鱼雷伤害 | 进水 debuff |
| LEVEL_BOMBER | 轰炸航线3阶段 | `panelDamage × 1.5` | 爆炸伤害 |
| RECON | — | 0（无攻击力） | — |

### 水平轰炸机航线模式

水平轰炸机采用3阶段轰炸航线循环：
1. **爬升阶段**：爬升到目标上方32格高度，重置所有航线状态
2. **投弹阶段**：锁定航向直线飞行，水平距离 < 3格时投弹，飞行速度 = `panelSpeed × 0.4`
3. **脱离阶段**：投弹后继续直飞10格，然后重新评估目标并回到阶段1

### 目标过滤

所有飞机的目标选择（`resolveTarget()`、`autoSeekTarget()`）排除水下目标（`isUnderWater()`）。

## 10. Buff系统

### 自定义效果

| Buff | 效果 | 代码位置 |

### 装填加速计算

```java
// Amplifier 0 = 80%冷却，Amplifier 1 = 60%冷却
float multiplier = 1.0f - (reloadBoost.getAmplifier() + 1) * 0.2f;
return Math.round(baseCooldown * multiplier);
```

### 高速规避减伤

```java
// Amplifier 0 = 20% 减伤，Amplifier 1 = 40% 减伤
float reduction = (evasion.getAmplifier() + 1) * 0.2f;
return damage * (1.0f - reduction);
```

### 经验提升
- 标记效果（Marker），实际 XP 加成在 `GameEvents.onXpDrop()` 中处理
- Level I (amplifier 0) = +50% XP

### 数据修改检查清单

修改任何数值时，检查以下关联数据：
- [ ] 负重是否与 `TransformationManager` 一致
- [ ] 冷却是否与 HUD 显示时间一致
- [ ] 伤害是否与 Buff 倍率相匹配
- [ ] 联装数是否与弹药消耗量一致
- [ ] 舰装容量是否足以放置所有装备
- [ ] 弹药配方是否需要同步更新
- [ ] 武器合成配方是否需要同步更新
- [ ] 深海NPC JSON 是否需要调整
- [ ] 配置项是否能影响此数值
- [ ] 翻译是否需要更新（zh_cn/en_us.json）
- [ ] 模组教程是否需要更新
