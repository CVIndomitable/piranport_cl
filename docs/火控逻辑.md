# 火控逻辑

> 覆盖版本：v0.0.11 Ruins（2026-04-18）。

本文档描述飞机、导弹、鱼雷、深弹四大武器系统的火控（锁定/瞄准/发射/制导）逻辑。

---

## 通用火控系统

### 锁定机制

| 按键 | 功能 | 说明 |
|------|------|------|
| P | 单体锁定 | 替换整个锁定列表为准心所指实体 |
| O | 追加锁定 | 将准心实体加入列表（上限4个） |
| I | 取消锁定 | 清空所有锁定 |
| H | 自动升空开关 | 同时控制战斗机自动起飞和防空导弹自动发射 |

- 锁定范围：80格（以视角射线检测）
- 服务端管理：`FireControlManager`（`Map<玩家UUID, List<目标UUID>>`）
- 客户端镜像：`ClientFireControlData`，由网络包同步
- 侦察机模式下，射线从侦察机位置发出（`Minecraft.setCameraEntity`）

### 冷却系统

所有武器共享一个基于舰装核心物品的逐槽冷却系统（`SlotCooldowns`），以背包槽位索引为 key。装备 `TorpedoReload` 增强件可缩短冷却。

### 友军伤害防护

所有弹丸均通过 `FriendlyFireHelper.shouldBlockHit()` 检查，跳过对同 `ownerUUID` 的玩家及其飞机的伤害。

### 两种操作模式

| 模式 | 条件 | 说明 |
|------|------|------|
| GUI模式 | 舰装核心在手持 | 通过核心GUI选择武器栏位发射 |
| 无GUI模式 | 核心在副手/背包 | 通过 `tryFireFromInventory()` 从任意快捷栏发射 |

火控锁定、自动升空、侦察机在两种模式下行为一致。

## 一、飞机系统

### 1.1 发射条件

- 燃油 > 0（`AircraftInfo.currentFuel()`）
- 载荷机型（轰炸机/鱼雷机）需在弹药槽有1个载荷物品，发射时消耗
- 生成位置：玩家眼睛高度 + 视线方向0.8格
- 发射后对应武器槽进入20tick冷却

### 1.2 飞行状态机

```
LAUNCHING → CRUISING → ATTACKING → RETURNING → REMOVED
                ↓
          RECON_ACTIVE（仅侦察机）
```

#### LAUNCHING（起飞）
- 爬升至 owner.Y + 18 格（巡航高度）
- 速度为面板速度的30%
- 到达目标高度（误差1.5格）或30tick后退出
- 侦察机直接进入 `RECON_ACTIVE`，其他进入 `CRUISING`

#### CRUISING（巡航）
- 以8格半径绕 owner 环绕飞行
- 角速度 = `panelSpeed × 0.015`/tick，速度30%
- FOLLOW 编组模式下改为绕侦察机飞行
- 进入攻击条件：火控锁定列表非空且至少1个目标存活
- 自动搜索：首次起飞时扫描32格内敌人，有则自动进入攻击

#### ATTACKING（攻击）
按机型分别描述（见下方1.3节）。

#### RETURNING（返航）
- 以40%速度直飞 owner 眼睛位置
- 距离 < 3格时回收，飞机物品归还原槽位（燃油清零需重新加油）

#### 强制返航触发条件
- 燃油耗尽
- 飞行超10分钟（12000tick）
- 卡住（120tick内移动 < 0.1格，连续两次）
- 超出距离限制（普通48格，侦察/FOLLOW模式200格）

#### 被击落
- HP降为0时爆炸粒子，不返还物品
- 掉落 `AircraftDropEntity`（浮力防爆实体），内含飞机物品可拾取

### 1.3 各机型攻击逻辑

#### 战斗机（Fighter）

| 参数 | 值 |
|------|------|
| HP | 20 |
| 攻击距离 | ~11格（±3格浮动） |
| 射速 | 每5tick一发 `BulletEntity` |
| 子弹伤害 | `panelDamage / 8` |
| 子弹速度/射程 | 2.5 / 24格 |

**目标获取优先级**：
1. 火控锁定目标
2. 32格内敌方 `AircraftEntity`
3. 32格内敌对怪物（Monster）

**空对空限定模式**（默认开启，H键切换）：
- 开启时自动搜索仅锁定飞机/飞行目标
- 火控手动锁定可覆盖此限制，锁任意实体

**弹药系统**（可选，config控制）：
- 启用后有弹药上限；耗尽尝试从核心弹药槽自动补给，无弹则返航

#### 俯冲轰炸机（Dive Bomber）

| 参数 | 值 |
|------|------|
| HP | 15 |
| 炸弹伤害 | `panelDamage × 1.5` |
| 爆炸范围 | 2.5 |

**攻击流程**：
1. **爬升**：升至目标上方18格，40%速度趋近，最多80tick
2. **提前量计算**：估算飞行时间 `t = 距离 / (速度×0.6)`，投弹点 = 目标位置 + 目标速度×t，锁定后不再修正
3. **俯冲**：60%速度直冲投弹点
4. **投弹**：水平距离 < 4格或高度低于投弹点时释放 `AerialBombEntity`（释放速度：0.2水平，-0.3垂直）
5. 投弹后返航（或自动补给后重新攻击）

#### 水平轰炸机（Level Bomber）

| 参数 | 值 |
|------|------|
| HP | 12 |
| 轰炸高度 | 目标上方32格 |
| 炸弹伤害 | `panelDamage × 1.5` |
| 爆炸范围 | 2.5 |

**攻击流程**：
1. **爬升**：升至目标上方32格
2. **建立航线**：锁定从当前位置到目标的直线方向，后续不再修正
3. **航线飞行**：沿航线方向匀速飞行（40%速度），保持轰炸高度
4. **投弹**：水平距离 < 3格时投下 `AerialBombEntity`（近乎垂直坠落：0.1水平，-0.1垂直）
5. **飞过目标**：继续飞行10格
6. **重新定位**：重置航线状态，循环下一轮（直到弹药耗尽）

#### 鱼雷机（Torpedo Bomber）

| 参数 | 值 |
|------|------|
| HP | 15 |
| 投放高度 | 目标上方2格 |
| 投放距离 | 20~30格 |

**攻击流程**：
1. 低空趋近目标（目标Y+2高度）
2. 水平距离进入20~30格区间时投下 `TorpedoEntity`（纯自由落体，初速(0, -0.6, 0)，`airDrop=true`）
3. 鱼雷落水后切换为水面巡航模式，方向为飞机的进入方向
4. 距离 < 20格时飞离重新进入；> 30格时低空接近
5. 200tick超时或投放后返航

#### 反潜机（ASW）

| 参数 | 值 |
|------|------|
| HP | 12 |
| 轰炸高度 | 目标上方20格 |
| 深弹伤害 | `panelDamage × 1.5` |
| 爆炸范围 | 3.0 |
| 投放间隔 | 30tick |

**攻击流程**：类似水平轰炸机，但高度更低（+20格）。水平距离 < 4格时投下 `DepthChargeEntity`，循环直到弹药耗尽。

**声呐探测**：巡航和攻击状态下每20tick扫描16格球形范围内水下/水生实体，通过 `AswSonarSyncPayload` 同步到客户端高亮显示（黄色描边）。

**反潜目标过滤**（`isAswTarget()`）：
- 深海潜艇（`DeepOceanSubmarineEntity`）
- 水生动物（`WaterAnimal`）
- 守卫者（`Guardian`）
- 当前在水下的任意怪物

#### 侦察机（Recon）

| 参数 | 值 |
|------|------|
| HP | 10 |
| 速度 | 面板速度的50% |
| 最大距离 | 200格 |

- 不进行攻击
- 玩家视角切换到飞机（`Minecraft.setCameraEntity`）
- WASD/空格/潜行控制飞机移动（每2tick发送 `ReconControlPayload`）
- 加速惯性 lerp=0.12，减速惯性 lerp=0.25
- 玩家本体获得最高级缓慢（等级9）防止移动
- 强制加载飞机周围区块（最多5区块半径）
- 背包中地图每20tick从飞机位置更新
- V键退出 / 超距200格退出 / 燃油耗尽退出

### 1.4 目标分配模式

通过编组数据（`FlightGroupData`）设置：

| 模式 | 行为 |
|------|------|
| FOCUS | 编组内所有飞机攻击锁定列表中第一个存活目标 |
| SPREAD | 每架飞机选择距自己最近的未被认领的锁定目标 |
| FOLLOW | 飞机绕侦察机环绕而非玩家，攻击时仍按火控锁定 |

### 1.5 转弯半径限制

`applyTurnRadiusConstraint`：每tick最大水平偏转角 = `速度 / TURN_RADIUS`（TURN_RADIUS=2.0格）。同时强制最低水平速度 = `panelSpeed × 0.05`，防止空中悬停。

## 二、导弹系统

### 2.1 导弹类型

| 类型 | 初速 | 加速/tick | 最高速 | 制导 | 命中效果 |
|------|------|-----------|--------|------|----------|
| ANTI_SHIP（反舰） | 0.04 | +0.04 | 2.4 | 追踪12°/tick | 穿甲直伤，无爆炸 |
| ANTI_AIR（防空） | 0.05 | +0.05 | 3.0 | 追踪12°/tick | 爆炸伤害 |
| ROCKET（火箭） | 0.04 | +0.04 | 2.0 | 无制导 | 爆炸伤害 |

所有导弹无重力，最大存活600tick（30秒）。

### 2.2 发射方式

#### 手动装填模式（反舰/火箭）
- 发射器需在装填设施（`ReloadFacilityBlock`）预装 `LOADED_AMMO`
- 每次 use() 发射1枚，扣减装填数
- 无发射间冷却，可速射至弹药耗尽

#### 自动装填模式（防空）
- 每次发射消耗背包中1个对应弹药物品
- 每次发射后进入逐槽冷却（`cooldownTicks`）
- 装备 `TorpedoReload` 增强件可缩短冷却

#### 生成参数
- 位置：玩家眼睛 + 视线方向0.5格，Y偏移-0.1
- 初速：视线方向 × 对应初速值

### 2.3 制导逻辑（反舰/防空）

每tick执行：
1. 若追踪目标为空且存有UUID，尝试从 `ServerLevel` 重新解析（容灾处理）
2. 若目标已死亡，清除并允许重新搜索
3. 若无目标且搜索冷却 ≤ 0：调用 `findTarget()`，冷却重置5tick
4. 有目标时调用 `homeToward(target)`

#### 追踪转向（homeToward）
- 每tick最大偏转 **12度**
- 目标点为实体中心（Y中点）
- 使用线性插值计算偏转：`t = maxTurnRad / 当前角度差`
- 方向修正后按当前速度重设运动矢量

### 2.4 目标获取（findTarget）

**优先级**：
1. **火控锁定**：列表中第一个存活、非容器、非水下实体
2. **自动搜索**（32格球形）：最近的存活敌对怪物

**类型限制**：
- 反舰/防空均跳过水下实体
- **防空额外跳过 `onGround()` 实体**——仅自动锁定空中目标；地面目标必须手动火控锁定

### 2.5 命中判定

| 类型 | 命中实体 | 命中方块 |
|------|----------|----------|
| 反舰 | 穿甲：临时扣减目标护甲→`thrown`伤害→恢复护甲。无穿甲：直接`thrown`伤害。无爆炸 | 静默消失 |
| 防空 | `explosion`伤害 + `level.explode()` | 爆炸（不破坏方块） |
| 火箭 | `explosion`伤害 + `level.explode()` | 爆炸（不破坏方块） |

命中/击杀/失败均通过聊天消息通知发射者。

### 2.6 防空导弹自动发射

由 `GameEvents.tickAutoLaunchFighters` 驱动，每40tick检查一次。

**触发条件**：
- 舰装核心 `SHIP_AUTO_LAUNCH` 数据组件为开（H键切换）
- 32格内存在存活、非水下、非地面的怪物

**执行流程**：
1. 扫描快捷栏0-8寻找防空型 `MissileLauncherItem`
2. 检查逐槽冷却
3. 从背包消耗1个弹药
4. 调用 `spawnMissileAutoAim()`
5. 应用冷却
6. 每次检查只发射1枚

**自动瞄准优先级**：
1. 火控锁定中第一个非水下目标
2. 32格内最近的空中怪物（跳过地面/水下）
3. 兜底：向上偏转发射（`max(lookY, 0.5)`）

## 三、鱼雷系统

### 3.1 鱼雷类型

| 口径 | 默认伤害 | 默认速度 | 存活时间 | 爆炸范围 |
|------|----------|----------|----------|----------|
| 533mm | 18 | 1.2 | 1200tick (60s) | 2.0 |
| 610mm | 28 | 1.0 | 1200tick (60s) | 2.5 |

每种命名鱼雷（共15种：G7a/G7e/MK系列/九式/零式等）可有独立属性覆盖默认值。

**特殊能力标签**（由 `TorpedoItem` 组件决定）：

| 标签 | 效果 |
|------|------|
| `magnetic` | 磁性近炸引信 |
| `wireGuided` | 有线制导（玩家操控） |
| `acoustic` | 声导鱼雷（自动追踪） |

### 3.2 发射方式

#### 手动装填模式
- 发射器需在装填设施预装 `LOADED_AMMO`
- 一次发射 `tubeCount` 枚

#### 自动装填模式
- 需装备 `TorpedoReload` 增强件
- 从背包消耗对应口径鱼雷 × `tubeCount`
- 弹药不足下一轮齐射时进入10tick惩罚冷却

#### 扇形散布
| 管数 | 散布角度 |
|------|----------|
| 1 | 0° |
| 2 | -3°, +3° |
| 3 | -4°, 0°, +4° |
| 4 | -6°, -2°, +2°, +6° |

- 以玩家视线方向为中心，水平旋转对应角度
- 生成位置：玩家眼睛 + 方向0.5格，Y偏移-0.3
- 初速：方向 × 鱼雷速度，Y=0（水面发射）
- 每次齐射消耗发射器1点耐久

### 3.3 飞行逻辑

每tick执行（按优先级顺序）：

#### 1. 空投入水（airDrop模式）
- 鱼雷机投下时启用
- 空中：自由落体（0.98阻力 + 0.08重力/tick）
- 检测到水面：切换为水面巡航，方向 = 飞机进入方向，速度 = 鱼雷速度

#### 2. 寿命倒计时
- 归零时在当前位置爆炸（`explosionRadius`）后消失

#### 3. 磁性近炸（magnetic）
- tickCount > 5 后生效
- 扫描3格球形范围内存活可选取非友军实体
- 检测到即触发 `magneticDetonate()`（爆炸）

#### 4. 有线制导（wireGuided）
- 玩家按9/0键发送 `TorpedoSteerPayload`
- 每次操作水平偏转 ±8度
- 最大线缆长度16格（超出自动断线，失去制导）

#### 5. 声导追踪（acoustic）
- tickCount > 10 后启用
- 扫描25格球形范围内存活可选取 `LivingEntity`
- 潜行实体探测距离缩减至10格
- 同时潜行且静止（速度 < 0.0001）的实体完全隐形
- 每tick最大偏转 **8度** 朝向最近目标

#### 6. 速度维持
- 每tick重新缩放水平分量至 `torpedoSpeed`，对抗0.99阻力衰减

#### 7. 水面贴附AI
| 状态 | 行为 |
|------|------|
| 空中且下方有水 | 清除Y速度 → 贴水面滑行 |
| 水中 | Y速度 +0.04/tick → 浮向水面 |
| 空中且下方无水 | Y速度 -0.06/tick → 下坠 |

### 3.4 命中判定

| 情形 | 效果 |
|------|------|
| 命中实体（非磁性） | `thrown`伤害 + 对 LivingEntity 施加进水效果（`FLOODING`，60tick，1魔法伤害/秒持续3秒） |
| 命中实体（磁性） | `explosion`伤害 + 爆炸 |
| 命中水中方块 | 静默消失 |
| 命中非水方块 | 爆炸（磁性用 `magneticDetonate()`，非磁性用 `level.explode()`） |

方块破坏由配置 `EXPLOSION_BLOCK_DAMAGE` 控制。

### 3.5 鱼雷无主动制导

标准鱼雷**完全无制导**——沿发射方向直线前进。玩家必须通过视角方向和扇形散布手动瞄准。有线制导允许发射后操控，声导为自动追踪，两者均为特定鱼雷物品的特殊能力。

## 四、深弹系统

### 4.1 深弹投射器型号

| 投射器 | 投弹数 | 冷却 | 散布模式 | 负重 |
|--------|--------|------|---------|------|
| 基础深弹投射器 | 1 | 60 tick | SINGLE | 2 |
| 改良深弹投射器 | 2 | 80 tick | FRONT_BACK | 3 |
| 高级深弹投射器 | 3 | 100 tick | TRIANGLE | 5 |

- 独立 `DEPTH_CHARGE` 武器分类，不与炮/鱼雷互斥
- 弹药：`depth_charge` 物品
- 冷却可被 `ReloadBoostEffect` 缩短

### 4.2 发射方式

玩家发射走 `ShipCoreItem.fireDepthCharges()`（GUI/无GUI 模式均从背包扫描弹药）：

```
玩家按下火控键 P
  → 检查背包(+副手) depth_charge 数量 ≥ chargeCount
  → 按 SpreadPattern 生成实体（每个 DepthChargeEntity 伤害14f，爆炸力3.0f）
      SINGLE      → 前方0.6速1枚
      FRONT_BACK  → 前方0.7速 + 0.4速各1枚
      TRIANGLE    → 中0.7速 + 左右±20°各0.5速
  → 所有实体初始速度包含 +0.3 的 Y 分量（轻微抛物）
  → 扣减投射器耐久 -1（耐久归零销毁）
  → 设置武器槽冷却
```

### 4.3 飞行与近炸

| 阶段 | 行为 |
|------|------|
| 入水前 | 正常弹道（轻抛 + 原版重力） |
| 入水后 | 下沉，`ARM_TICKS=5` 后武装 |
| 近炸 | 检测 3.0 格内 LivingEntity 立即引爆 |
| 触底/超时 | `MAX_LIFETIME=600 tick` 或触地触发爆炸 |

### 4.4 伤害判定

- 水平伤害半径 `BLAST_RADIUS=5.0` 格，垂直容差 `BLAST_HEIGHT=2.0` 格
- 伤害类型：`magic`（魔法伤害，无视护甲）
- 伤害量：`damage × (1 - distance / BLAST_RADIUS)`（线性衰减）
- 友军伤害防护：通过 `FriendlyFireHelper` 过滤发射者及其飞机

### 4.5 反潜机深弹

反潜机（ASW）在攻击阶段投放 `DepthChargeEntity`，机制见第1章"1.3 反潜机"。声呐探测在巡航+攻击状态下扫描 16 格球形。
