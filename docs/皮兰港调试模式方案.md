# 皮兰港调试模式方案

## 概述

模组内置调试系统，游戏内快捷键一键开关，输出独立日志到 `logs/piranport-debug.log`。主要面向 Claude Code 分析 bug 和性能问题。

---

## 操作方式

| 操作 | 效果 |
|------|------|
| `F8` | 切换调试模式开关，屏幕左上角显示/隐藏 `[PP DEBUG]` 水印 |
| `Shift+F8` | 立即输出当前状态快照到日志（不需要开启调试模式） |

- 默认关闭，不影响正常游戏
- 仅客户端按键，但通过 C2S 包通知服务端同时开启服务端日志
- 关闭时零开销（所有埋点第一行就是 `if (!enabled) return;`）

---

## 日志输出

### 文件位置

`logs/piranport-debug.log`，每次启动游戏时清空重建。

### 日志格式

```
[12:34:56.789] [PERF] WeightScan | 340ns | player=Steve weight=46/72
[12:34:57.012] [EVENT] Transform ON | player=Steve core=medium_ship_core weapons=[small_gun,small_gun,twin_torpedo_launcher] weight=20/72
[12:34:58.100] [EVENT] Reload START | weapon=small_gun slot=0 ammo=small_he_shell
[12:34:59.600] [EVENT] Reload DONE | weapon=small_gun slot=0 elapsed=30tick
[12:35:01.200] [EVENT] Fire | weapon=small_gun ammo=small_he_shell remaining=14
[12:35:03.400] [EVENT] Evasion DODGE | chance=15% roll=0.08 source=zombie
[12:35:10.000] [EVENT] Aircraft LAUNCH | type=FIGHTER group=1 ammo=fighter_ammo fuel=1200
[12:35:15.000] [ERROR] Aircraft ORPHAN | entityId=1847 reason=owner_offline
```

每行固定格式：`[时间戳] [级别] 标签 | 键值对`，方便 Claude Code 用 grep/awk 解析。

## 埋点清单

### PERF（性能计时）

| 标签 | 采样位置 | 说明 |
|------|---------|------|
| `WeightScan` | 背包负重计算 | 每次触发时记录耗时 |
| `ReloadTick` | TransformationManager 装填逻辑 | 每 100 tick 采样一次 |
| `FireControlResolve` | AircraftEntity.resolveTarget() | 每次目标解析 |
| `AircraftTick` | AircraftEntity.tick() | 按状态分类记录 |
| `EvasionCheck` | LivingHurtEvent 闪避判定 | 每次触发 |

### EVENT（关键事件）

| 标签 | 触发点 | 记录内容 |
|------|--------|---------|
| `Transform ON/OFF` | 变身/解除 | 核心类型、武器列表、负重 |
| `Reload START/DONE` | 装填开始/完成 | 武器、弹药类型、耗时 |
| `Fire` | 武器发射 | 武器类型、弹药、剩余数量 |
| `Aircraft LAUNCH/RETURN/KILLED` | 飞机生命周期 | 类型、编组、弹药、燃料 |
| `FireControl LOCK/ADD/CANCEL` | 火控操作 | 目标 UUID、名称、距离 |
| `Evasion DODGE/HIT` | 闪避触发 | 概率、骰子值、伤害来源 |
| `ReloadBoost ACTIVE` | 装填加速生效 | 等级、倍率 |
| `Buff APPLY/REMOVE` | 自定义 Buff 变化 | 效果类型、等级、来源食物 |
| `FlightGroup UPDATE` | 编组配置变更 | 组号、成员、弹种、模式 |

### ERROR（异常）

| 标签 | 触发点 | 记录内容 |
|------|--------|---------|
| `Aircraft ORPHAN` | 孤儿实体检测 | 实体 ID、原因 |
| `DataComponent FAIL` | DataComponent 读取异常 | 物品、组件类型、异常信息 |
| `Network FAIL` | 网络包异常 | 包类型、异常信息 |
| `ItemReturn FAIL` | 飞机物品返还失败 | 槽位、物品、原因 |
| `WeightOverflow` | 负重超限 | 当前负重、上限、超出物品 |

## 实现结构

```java
public final class PiranPortDebug {
    private static boolean enabled = false;
    private static final Logger DEBUG_LOGGER = LogManager.getLogger("PiranPortDebug");
    
    // --- 开关 ---
    public static void toggle() {
        enabled = !enabled;
        if (!enabled) writeSummary();
        stats.reset();
    }
    
    public static boolean isEnabled() { return enabled; }
    
    // --- 日志 ---
    public static void perf(String tag, long nanos) {
        if (!enabled) return;
        DEBUG_LOGGER.info("[PERF] {} | {}ns | {}", tag, nanos, context);
        stats.recordPerf(tag, nanos);
    }
    
    public static void event(String msg, Object... args) {
        if (!enabled) return;
        DEBUG_LOGGER.info("[EVENT] " + msg, args);
        stats.recordEvent(msg);
    }
    
    public static void error(String msg, Object... args) {
        if (!enabled) return;
        DEBUG_LOGGER.error("[ERROR] " + msg, args);
        stats.recordError();
    }
    
    // --- 快照（不依赖 enabled）---
    public static void snapshot(ServerPlayer player) {
        DEBUG_LOGGER.info(buildSnapshot(player));
    }
    
    // --- 摘要 ---
    private static void writeSummary() {
        DEBUG_LOGGER.info(stats.format());
    }
    
    // --- 统计累加器 ---
    private static final DebugStats stats = new DebugStats();
}
```

### 按键注册

```java
// ModKeyMappings.java
public static final KeyMapping DEBUG_TOGGLE = new KeyMapping(
    "key.piranport.debug_toggle", GLFW.GLFW_KEY_F8, "key.categories.piranport");

// ClientTickHandler.java
if (DEBUG_TOGGLE.consumeClick()) {
    if (Screen.hasShiftDown()) {
        // Shift+F8: 发C2S包请求服务端输出快照
        sendSnapshotRequest();
    } else {
        // F8: 切换调试开关，发C2S包同步服务端
        PiranPortDebug.toggle();
        sendDebugToggle(PiranPortDebug.isEnabled());
    }
}
```

### Log4j2 配置

在 `src/main/resources/log4j2.xml` 中追加（或新建模组专用配置）：

```xml
<Appender type="File" name="PiranPortDebug" fileName="logs/piranport-debug.log" append="false">
    <PatternLayout pattern="[%d{HH:mm:ss.SSS}] %msg%n"/>
</Appender>
<Logger name="PiranPortDebug" level="INFO" additivity="false">
    <AppenderRef ref="PiranPortDebug"/>
</Logger>
```

`append="false"` 表示每次启动清空，`additivity="false"` 表示不写入主日志。

## 调用示例

### 性能埋点

```java
// TransformationManager.java
public void recalculateWeight(Player player) {
    long t = System.nanoTime();
    // ... 扫描逻辑 ...
    PiranPortDebug.perf("WeightScan", System.nanoTime() - t);

```

### 事件埋点

```java
// ShipCoreItem.java
public void onTransform(Player player, boolean transformOn) {
    // ... 变身逻辑 ...
    PiranPortDebug.event("Transform {} | player={} core={} weight={}/{}",
        transformOn ? "ON" : "OFF", player.getName().getString(),
        coreId, currentWeight, weightLimit);
}
```

### Claude Code 使用场景

```bash
# 查看最新快照
cat logs/piranport-debug.log | grep "Snapshot" -A 30

# 查看性能统计
cat logs/piranport-debug.log | grep "\[PERF\]"

# 查看所有错误
cat logs/piranport-debug.log | grep "\[ERROR\]"

# 查看会话摘要
tail -20 logs/piranport-debug.log

# 查看闪避触发记录
cat logs/piranport-debug.log | grep "Evasion"

# 查看飞机生命周期
cat logs/piranport-debug.log | grep "Aircraft"
```

## 适用版本

调试系统与版本无关，可在任意版本插入。建议与 v0.0.5 一起实现（Phase 25 之前作为 Phase 25a，或作为独立的基础设施 Phase）。后续每个新功能开发时顺手加埋点即可。
