## 状态机总览

所有飞机共享同一状态机，6种状态：

```
                ┌──────────────────────────────────┐
                │           LAUNCHING              │
                │      (30 tick 爬升期)             │
                └──────────┬───────────┬───────────┘
                           │           │
                  非侦察机 │           │ 侦察机
                           ▼           ▼
                ┌──────────────┐  ┌────────────────┐
                │   CRUISING   │  │  RECON_ACTIVE  │
                │  (巡航环绕)  │  │ (玩家操控侦察) │
                └──────┬───────┘  └───────┬────────┘
                       │                  │
            发现火控目标│                  │ 召回/超距/登出
                       ▼                  │
                ┌──────────────┐          │
                │  ATTACKING   │          │
                │ (各型号攻击) │          │
                └──────┬───────┘          │
                       │                  │
          弹药耗尽/任务│完成               │
                       ▼                  ▼
                ┌──────────────────────────────────┐
                │           RETURNING              │
                │    (飞回主人，速度×0.4)           │
                └──────────────┬───────────────────┘
                               │
                          距离<3格│
                               ▼
                ┌──────────────────────────────────┐
                │           REMOVED                │
                │     (回收物品，实体移除)          │
                └──────────────────────────────────┘
```

> ATTACKING 与 CRUISING 可双向切换：目标全部死亡时回到巡航，新目标出现时重新进入攻击。

## 通用生命周期

### 1. 发射

- 入口：`AircraftItem.use()` 或 `ShipCoreItem.tryFireFromInventory()`
- 在主人眼前方向生成 `AircraftEntity`
- 初始状态：`LAUNCHING`
- 保存原始 ItemStack（用于回收时还原）

### 2. 起飞 (LAUNCHING)

- 持续 **30 tick**
- 目标高度：主人 Y + 18 格（巡航高度）
- 爬升速度：`panelSpeed × 0.3`
- 水平移动阻尼 ×0.5（压制横漂）
- 到达高度或超时 → 非侦察机进入 `CRUISING`，侦察机直接进入 `RECON_ACTIVE`

### 3. 巡航 (CRUISING)

- 围绕主人以 **8格半径** 环绕飞行，高度 18 格
- 环绕速度：`panelSpeed × 0.3`
- 每 4 tick 消耗 1 燃油
- FOLLOW 模式下：改为环绕主人的侦察机
- 定期清理火控列表中已死目标
- 检测到火控目标（且存活）→ 转入 `ATTACKING`

### 4. 攻击 (ATTACKING)

- 最少 20 tick 后才可回退 CRUISING（防抖动）
- 各机型有独立攻击逻辑（见下文各节）
- 持续消耗燃油

### 5. 返航 (RETURNING)

- 速度：`panelSpeed × 0.4`，朝主人眼部飞行
- 距离 < 3 格 → 触发回收

### 6. 回收 (REMOVED)

- 还原飞机物品到原武器槽（满则掉落）
- **燃油重置为 0**（必须再次加油才能起飞）
- 弹药保留当前剩余量
- 侦察机额外清理：释放区块加载、移除减速效果、通知客户端恢复摄像机

### 7. 被击落

- 血量归零 → 不返还物品
- 生成 `AircraftDropEntity`（防爆、水面浮动）掉落在原地
- 通知主人"xxx 被 yyy 击落"

## 战斗机 (FIGHTER)

**角色**：空战/对地扫射，子弹攻击

```
CRUISING ──火控目标──→ ATTACKING
                         │
            ┌────────────┤
            │   保持11格距离
            │   每5 tick射击
            │            │
            │     弹药耗尽(开启时)
            │            │
            ▼            ▼
         CRUISING    RETURNING
        (目标死亡)   (弹药耗尽)
```

### 攻击细节

| 参数 | 值 |
|------|------|
| 首选距离 | 11 格 |
| 射击间隔 | 5 tick |
| 射程 | 24 格 |
| 单发伤害 | `panelDamage / 8` |
| 弹药容量 | 64 发（仅 fighterAmmoEnabled=true 时消耗） |
| 子弹重力 | 0.005（极低，近似直线） |

### 距离控制

- **> 14 格**：以 `panelSpeed × 0.5` 接近
- **< 8 格**：以 `panelSpeed × 0.2` 后退拉开
- **8~14 格**：减速到当前速度的 80%

### 目标类型

- 火控锁定的任何实体（含飞机）
- 自动搜索：优先敌方飞机 → 地面怪物（32格范围，仅一次）

## 俯冲轰炸机 (DIVE_BOMBER)

**角色**：高空俯冲精确投弹，单次攻击

```
CRUISING ──火控目标──→ ATTACKING
                         │
              Phase 1: 爬升到目标+18格
                         │
                 到达高度 or 80 tick 超时
                         │
              Phase 2: 提前量计算，锁定投弹点
                         │
              Phase 3: 俯冲，速度×0.6
                         │
                 距离<4 or 高度低于投弹点
                         │
                    ★ 投弹 ★
                         │
                   ┌─────┴─────┐
                   │           │
              自动补给ON    自动补给OFF
                   │           │
                CRUISING    RETURNING
               (续战)      (任务完成)
```

### 攻击细节

| 参数 | 值 |
|------|------|
| 爬升高度 | 目标 Y + 18 格 |
| 爬升超时 | 80 tick |
| 俯冲速度 | `panelSpeed × 0.6` |
| 炸弹伤害 | `panelDamage × 1.5` |
| 爆炸半径 | 2.5 格 |
| 炸弹重力 | 0.06（强，模拟自由落体） |
| 弹药 | 每次 1 枚 |

### 提前量预判

俯冲锁定时计算目标移动预判：
```
预估到达时间 = 当前距离 / 俯冲速度
预判位置 = 目标位置 + 目标速度 × 预估到达时间
```
一旦锁定，投弹点不再重新计算（committed dive）。

## 鱼雷机 (TORPEDO_BOMBER)

**角色**：低空接近，投放鱼雷

```
CRUISING ──火控目标──→ ATTACKING
                         │
                  ┌──────┴──────┐
                  │             │
             距离<20格      距离>30格
             (太近)         (太远)
                  │             │
             反向拉开      低空接近
             (速度×0.4)    (高度=目标+2格)
                  │             │
                  └──────┬──────┘
                         │
                    距离 20~30 格
                         │
                    ★ 投放鱼雷 ★
                    (垂直入水→水平巡航)
                         │
                   ┌─────┴─────┐
                   │           │
              自动补给ON    自动补给OFF
                   │           │
                CRUISING    RETURNING
```

### 攻击细节

| 参数 | 值 |
|------|------|
| 飞行高度 | 目标 Y + 2 格（低空） |
| 投放区间 | 水平 20~30 格 |
| 攻击超时 | 200 tick |
| 鱼雷初速 | 垂直 -0.6（空投入水） |
| 弹药 | 每次 1 枚 |

### 鱼雷空投机制

1. 鱼雷以垂直姿态（dy=-0.6）投下
2. 设置 `airDrop=true`，同时传入水平方向
3. 鱼雷入水后转为水平巡航，按预定方向前进

## 水平轰炸机 (LEVEL_BOMBER)

**角色**：高空水平轰炸，可多次航线投弹

```
CRUISING ──火控目标──→ ATTACKING
                         │
              Phase 1: 爬升到目标+32格
                         │
              Phase 2: 锁定航线方向(直线)
                 沿航线直飞，速度×0.4
                         │
                 飞越目标上方(水平<3格)
                         │
                    ★ 投弹 ★
                         │
              Phase 3: 继续飞行10格
                         │
                    重置航线状态
                         │
                  ┌──────┴──────┐
                  │             │
              目标存活       弹药耗尽
                  │             │
            回到Phase 1    RETURNING
            (重新进入)
```

### 攻击细节

| 参数 | 值 |
|------|------|
| 轰炸高度 | 目标 Y + 32 格 |
| 航线速度 | `panelSpeed × 0.4` |
| 炸弹伤害 | `panelDamage × 1.5` |
| 爆炸半径 | 2.5 格 |
| 投弹条件 | 水平距离 < 3 格 |
| 飞越距离 | 投弹后继续飞 10 格再重置 |

### 轰炸航线机制

- 到达高度后锁定一条**直线航线方向**（从当前位置指向目标）
- 航线一旦锁定不改变，直到本轮投弹+飞越完成
- 高度修正：`(目标高度 - 当前高度) × 0.15`，保持平稳
- 投弹后炸弹继承少量水平速度（×0.1），主要靠重力下落
- 飞越 10 格后清空航线状态，下一 tick 在 `tickAttacking` 中重新选择目标

### 与俯冲轰炸机的区别

| | 俯冲轰炸机 | 水平轰炸机 |
|---|---|---|
| 高度 | 目标+18 | 目标+32 |
| 投弹方式 | 俯冲后投弹 | 水平飞越投弹 |
| 提前量 | 有（预判落点） | 无（靠飞越正上方） |
| 投弹后 | 直接返航/补给 | 继续飞10格，可多次航线循环 |
| 单次/多次 | 单次 | 多次（目标存活则重复） |

## 反潜机 (ASW)

**角色**：反潜作战，投放深弹，附带声纳扫描

```
CRUISING ──火控目标(水下)──→ ATTACKING
   │                           │
   │(每20tick声纳扫描)   爬升到目标+20格
   │                           │
   │                     飞到目标正上方
   │                     (水平<4格)
   │                           │
   │                    ★ 投放深弹 ★
   │                    (冷却30 tick)
   │                           │
   │                     目标存活→继续环绕投弹
   │                     弹药耗尽→RETURNING
```

### 攻击细节

| 参数 | 值 |
|------|------|
| 攻击高度 | 目标 Y + 20 格 |
| 投弹间隔 | 30 tick |
| 深弹伤害 | `panelDamage × 1.5` |
| 近炸半径 | 3 格（5 tick 后解锁） |
| 爆炸范围 | 水平 5 格，垂直 ±2 格（扁柱形） |
| 伤害类型 | 魔法伤害（无视护甲） |
| 伤害衰减 | 中心 100% → 边缘 50% |

### 声纳扫描

- 巡航和攻击期间每 20 tick 执行
- 扫描范围：以飞机为中心 16 格
- 检测水下实体（水中生物 + 潜艇 + 水下怪物）
- 最多 128 个检测结果
- 通过 `AswSonarSyncPayload` 同步到客户端 HUD

### ASW 目标判定

以下实体视为反潜目标：
- `DeepOceanSubmarineEntity`（深海潜艇）
- `WaterAnimal`（水生动物）
- `Guardian`（守卫者）
- 任何在水中的 `Monster`

## 侦察机 (RECON)

**角色**：远距离侦察，玩家直接操控

```
LAUNCHING ──30 tick──→ RECON_ACTIVE
                          │
                   玩家操控飞行
                   (WASD输入)
                          │
               ┌──────────┼──────────┐
               │          │          │
           手动召回   超200格距离   主人登出
               │          │          │
               └──────────┼──────────┘
                          │
                      RETURNING
                          │
                       REMOVED
```

### 操控细节

| 参数 | 值 |
|------|------|
| 飞行速度 | `panelSpeed × 0.5` |
| 最小前进速度 | 速度的 15%（不能悬停） |
| 加速因子 | 0.12（惯性感） |
| 减速因子 | 0.25（响应快） |
| 最大距离 | 200 格（超出自动返航） |
| 血量 | 10 HP（最脆弱） |

### 区块加载

- 围绕飞机位置强制加载区块，半径 = min(服务器视距, 5)
- 飞机移动到新区块时差量更新（释放旧区块、加载新区块）
- 新区块以 16 块/tick 限速发送给客户端
- 侦察结束时释放所有强制区块

### 地图同步

- 每 20 tick 以飞机位置更新主人背包中的地图
- 等效于"携带地图在高空飞行"，可用于探索地形

### 其他飞机的 FOLLOW 模式

- 其他飞机设为 FOLLOW 攻击模式时，巡航阶段改为环绕侦察机飞行
- 跟随距离上限 200 格（与侦察机一致）
- 侦察机消失则回退到环绕主人

## 转向半径约束

所有飞机在每 tick 运动后都受转向半径限制。

### 原理

```
最大偏转角(弧度/tick) = 当前水平速度 / 转弯半径(2格)
```

- 计算上一 tick 水平方向与本 tick 期望方向的角度差
- 若超过最大偏转角 → 限制为最大角度的圆弧
- 垂直分量（Y速度）不受限制
- 起飞阶段（LAUNCHING）跳过此约束

### 防悬停

- 最低水平速度 = `panelSpeed × 0.05`
- 速度低于此值时沿上一 tick 方向强制补速
- 起飞阶段除外

### 效果

飞机无法瞬间掉头，必须画弧线转弯，模拟真实飞行器的转向特性。

## 目标选择逻辑

### 攻击模式

| 模式 | 巡航行为 | 目标选择 |
|------|---------|---------|
| **FOCUS** | 环绕主人 | 火控列表第一个存活目标 |
| **SPREAD** | 环绕主人 | 距飞机最近的存活目标 |
| **FOLLOW** | 环绕侦察机 | 同 FOCUS/SPREAD，但跟随范围 200 格 |

### 目标优先级

**战斗机**：
1. 火控锁定实体（任何类型，含飞机）
2. 自动搜索：32 格内敌方飞机 → 地面怪物（仅一次）

**攻击机（俯冲/鱼雷/水平轰炸）**：
1. 火控锁定的地面/水面目标（跳过空中目标）
2. 自动搜索：32 格内地面怪物（仅一次）

**反潜机**：
1. 火控锁定的 ASW 目标（潜艇/水生/水下怪物）
2. 自动搜索：32 格内 ASW 目标（仅一次）

**侦察机**：无攻击目标

### 自动搜索规则

- 触发条件：无火控锁定 + 从未有过火控 + 尚未搜索过
- 范围：32 格
- 仅执行一次（`autoSeekDone` 标记）
- 若后来获得过火控目标，不再触发自动搜索

## 防御机制

| 机制 | 条件 | 动作 |
|------|------|------|
| 最大滞空时间 | 12,000 tick (10分钟) | 强制返航 |
| 卡住检测 | 连续 2 次检查（间隔60tick）移动 < 0.1 格 | 强制返航 |
| 超距（普通） | 距主人 > 48 格 | 传送回主人附近 + 强制返航 |
| 超距（FOLLOW/侦察） | 距主人 > 200 格 | 传送/结束侦察 + 强制返航 |
| 燃油耗尽 | 燃油 = 0 | 强制返航 |
| 无有效载荷 | payloadType 为空且无子弹 | 直接返航 |

> 卡住检测在 RECON_ACTIVE 期间禁用（玩家可能故意悬停观察）。
