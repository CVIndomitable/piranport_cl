# 飞机模型方向核对结论

> 核对日期：2026-04-17
> 核对工具：`/ppd model_debug <model>`（见 `docs/命令.md`）

---

## 核对方法

1. 开启调试模式（F8）
2. 站到空地，执行 `/ppd model_debug b25` 或 `/ppd model_debug f4f`
3. 中心方块渲染静止模型（等价于 `AircraftRenderer` 在 `entityYaw=0 / pitch=0` 的变换）
4. 读取机头、机腹指向的告示牌方向

---

## 结论

| 机型 | 机头 | 机腹 | 模型 Java 文件 |
|------|------|------|---------------|
| B-25 | **+Z（南）** | **-Y（下）** | `B25Model.java` |
| F4F  | **+Z（南）** | **-Y（下）** | `F4FModel.java` |

两套模型建模方向完全一致，`AircraftRenderer` 统一变换对两者都正确。

## `AircraftRenderer` 变换公式

当 `entityYaw = 0` 时，对模型局部坐标 `(x_l, y_l, z_l)` 做如下变换得到世界偏移：

```
poseStack.mulPose(Axis.YP.rotationDegrees(180.0f - entityYaw));  // 绕 Y 轴旋转
poseStack.mulPose(Axis.XP.rotationDegrees(-pitch));              // 绕 X 轴俯仰
poseStack.scale(-1.5f, -1.5f, 1.5f);                             // 翻转 X/Y、放大 1.5
poseStack.translate(0.0, -1.501, 0.0);
```

yaw=0、pitch=0 时的映射：

```
世界坐标 = ( x_local,  -(y_local - 1.501),  -z_local )
```

## Blockbench 建模约定（新机型请遵守）

为了能复用 `AircraftRenderer` 和 `/ppd model_debug`，新机型在 Blockbench 里建模时**必须**遵循：

- **机头朝 -Z 轴**（模型空间内）
- **机背朝 -Y 轴**（模型空间内，Blockbench 默认 Y 倒置）
- 模型 pivot 放在 `(0, 24, 0)`（entity root 标准位置）
- 机翼沿 ±X 展开即可（对称模型左右无严格约束）

建好之后，运行 `/ppd model_debug <新机型>` 核对，机头必须指向 SOUTH（+Z）、机腹指向 DOWN（-Y）。

## MC 实体朝向约定对照

| `entityYaw` | 机头世界朝向 |
|-------------|-------------|
| 0           | +Z（南） |
| 90          | -X（西） |
| 180         | -Z（北） |
| 270         | +X（东） |

面朝 +Z（yaw=0）时，飞行员右手 = `forward × up` = (0,0,1) × (0,1,0) = **(-1, 0, 0)** = -X。

**含义：** 任何"左/右翼偏移""右侧发射"之类的逻辑都应从 `entity.getLookAngle()` 推导右向量（`up × lookDir`），不要直接写死世界坐标轴。
