### 1.5 数值说明

- **伤害**：直接传入 `TorpedoEntity.setDamage()`，命中时作为 `damageSources().thrown()` 的 amount
- **航程**：原始单位，实际 lifetime = `range × 20` ticks（即 `range` 秒）
- **速度**：blocks/tick，每 tick 水平方向恒定以此速度前进（抵消原版 0.99 阻力后恢复到该值）

## 二、发射流程

### 2.1 玩家发射 — 手动装填模式 (`fireTorpedosManualMode`)

**前置条件**：
- 发射器已通过装填设施预装弹药（`LoadedAmmo` DataComponent）
- 装填数量 >= 发射器联装数 (`tubeCount`)
- 无需装备鱼雷装填器

**流程**：
```
玩家按下火控键 P
  → ShipCoreItem 检测武器槽为 TorpedoLauncherItem
  → 检查 LoadedAmmo.count() >= tubeCount
  → 解析已装填弹药 ID → 获取 TorpedoItem 实例
  → 读取 magnetic / wireGuided / acoustic 标志
  → 读取 damage / speed / lifetime 数值
  → getSpreadAngles(tubeCount) 计算扇形角度
  → 循环创建 TorpedoEntity：
      位置 = 玩家坐标 + 前方0.5格, 眼高-0.3
      速度 = rotateHorizontal(视线方向, 扇形角) × torpedoSpeed
      Y速度 = 0（水平发射）
  → 清空发射器 LoadedAmmo
  → 发射器耐久 -1（耐久归零则销毁）
  → 设置武器冷却
```

**扇形角度表**：

| 联装数 | 角度(度) |
|--------|----------|
| 1 | [0] |
| 2 | [-3, 3] |
| 3 | [-4, 0, 4] |
| 4 | [-6, -2, 2, 6] |

### 2.2 玩家发射 — 背包扫描模式 (`fireTorpedosInventoryMode`)

**前置条件**：
- 必须装备鱼雷装填器 (`TorpedoReloadItem`)
- 背包中有足量同口径鱼雷物品

**流程**：
```
玩家按下火控键 P
  → ShipCoreItem 检测武器槽为 TorpedoLauncherItem
  → hasTorpedoReloadEquipped → 走 InventoryMode
  → 扫描背包 + 副手（跳过核心槽和武器槽），统计同口径鱼雷数量
  → 数量 < tubeCount → 显示"弹药不足"，中止
  → 按顺序消耗鱼雷物品，记录第一个鱼雷的属性作为齐射参考
  → 任一鱼雷有 magnetic/acoustic/wireGuided 则整个齐射带该属性
  → 创建实体（同手动模式）
  → 发射器耐久 -1
  → 检查背包剩余弹药：
      足够下一轮 → 正常冷却
      不够 → 10 tick 短冷却 + "弹药不足"提示
```

**关键差异**：背包模式下，齐射属性由消耗到的所有鱼雷取并集（任一有磁性则全部磁性）。数值属性（伤害/速度/航程）取第一颗鱼雷的参数。

### 2.3 无GUI模式发射 (`fireTorpedos`)

逻辑与背包扫描模式基本一致，入口为 `ShipCoreItem.fireTorpedos()`，在无GUI模式的武器循环发射路径中调用。

### 2.4 鱼雷机空投

```
AircraftEntity (TORPEDO_BOMBER) 进入 ATTACKING 状态
  → tickTorpedoBomberAttack()
  → 目标距离 < 20: 远离目标，拉开距离
  → 目标距离 20~30: 投弹窗口
      → new TorpedoEntity (无 shooter 构造)
      → setOwner(owner)  // 归因到玩家
      → setDeltaMovement(0, -0.6, 0)  // 垂直下坠
      → setAirDrop(true, 指向目标方向)
      → setSourceAircraftName(飞机名)
  → 目标距离 > 30: 低空接近（高度 = 目标Y+2）
  → 投弹后 → 返航
```

### 2.5 深海NPC发射 (`TorpedoAttackGoal`)

```
canUse: mob.canUseTorpedoes() && 目标距离 <= 20格
每tick:
  → cooldown > 0 → 递减，等待
  → cooldown == 0 → fireTorpedoSalvo()
      → 3发扇形齐射，间隔8°
      → 速度 0.8 blocks/tick
      → 伤害 = mob 的 ATTACK_DAMAGE 属性值
      → 不设 magnetic/acoustic/wireGuided（通用直航）
      → 冷却重置 200 ticks (10秒)
```

## 三、航行逻辑 (`TorpedoEntity.tick()`)

### 3.1 tick 执行顺序

```
super.tick()  // ThrowableItemProjectile 移动+碰撞+0.99阻力+重力(=0)
  │
  ├── isRemoved? → 结束
  │
  ├── ① 航程递减 (--lifetime)
  │     └── <= 0 → 超时自爆 → discard
  │
  ├── ② 磁性近炸检测 (magnetic && tickCount > 5)
  │     └── checkMagneticProximity()
  │
  ├── ③ 线导掉线检测 (wireGuided)
  │     ├── 首次 tick → 记录 launchPos
  │     └── 距发射点 > 16格 → wireGuided = false, 通知玩家
  │
  ├── ④ 声导追踪 (acoustic && tickCount > 10)
  │     └── acousticHoming()
  │
  ├── ⑤ 空投阶段 (airDrop == true)
  │     ├── 着水 → airDrop = false, 切换水面巡航
  │     └── 空中 → 自由落体 (drag 0.98, gravity -0.08)
  │     └── return (跳过后续速度/水面逻辑)
  │
  ├── ⑥ 水平速度恢复
  │     └── 当前水平速度 → 缩放到 torpedoSpeed
  │
  └── ⑦ 水面贴合
        ├── 空气+下方是水 → Y速度=0 (水面航行)
        ├── 在水中 → Y速度+0.04 (上浮)
        └── 空中无水 → Y速度-0.06 (坠落)
```

### 3.2 速度恢复机制

原版 `ThrowableItemProjectile.super.tick()` 会施加 0.99 的水平阻力。鱼雷在每 tick 末尾将水平速度重新缩放到 `torpedoSpeed`，实现恒速航行：

```java
double currentH = sqrt(motion.x² + motion.z²);
if (currentH > 0.001) {
    scale = torpedoSpeed / currentH;
    motion = (motion.x * scale, motion.y, motion.z * scale);
}
```

这意味着鱼雷从发射到命中/超时，水平速度始终恒定。

### 3.3 水面贴合 AI

鱼雷没有使用原版重力（`getDefaultGravity() = 0.0`），而是根据当前方块状态手动控制垂直运动：

| 情况 | Y速度处理 | 效果 |
|------|-----------|------|
| 空气 + 下方水面 | Y = 0 | 贴着水面滑行 |
| 在水中 | Y += 0.04 | 缓慢上浮至水面 |
| 空中（下方无水）| Y -= 0.06 | 坠落 |

### 3.4 空投阶段

鱼雷机投下的鱼雷有独立的空投阶段：

1. 初始速度 `(0, -0.6, 0)` — 垂直下坠
2. 每 tick 施加 `(0.98x drag, -0.08 gravity, 0.98z drag)` — 模拟空中自由落体
3. 判定着水条件：`当前方块是水` 或 `当前方块是空气+下方是水`
4. 着水后 `airDrop = false`，速度切换为 `(direction.x × speed, 0, direction.z × speed)`
5. 后续进入正常水面巡航逻辑

## 四、特殊制导系统

### 4.1 磁性引信 (`magnetic`)

**原理**：在鱼雷周围 3 格范围内探测到可命中实体时直接引爆。

**参数**：
- 武装延迟：5 ticks（防止自爆）
- 引爆距离：3.0 格
- 爆炸半径：与普通鱼雷相同（533mm=2.0, 610mm=2.5）

**检测逻辑** (`checkMagneticProximity`):
```
扩展碰撞箱 3格 → 获取范围内实体
  过滤：非发射者 && 非友军 && 存活 && 可拾取
  任一实体距离 <= 3.0 → magneticDetonate()
```

**引爆** (`magneticDetonate`):
```
exploded = true
level.explode(爆炸半径, TNT或NONE取决于配置)
discard()
```

**命中实体时**：普通鱼雷用 `thrown` 伤害源，磁性鱼雷用 `explosion` 伤害源 + 触发爆炸。

### 4.2 线导制导 (`wireGuided`)

**原理**：玩家通过按键实时遥控鱼雷转向，导线超距自动断开。

**参数**：
- 最大线距：16 格（从发射点计算）
- 每次转向：8° (`WIRE_STEER_DEGREES`)
- 操作按键：9（左转）/ 0（右转），通过 `TorpedoSteerPayload` C2S包同步

**控制逻辑** (`steer`):
```java
// direction: -1=左, +1=右
double angleRad = toRadians(8° * direction);
newX = motion.x * cos - motion.z * sin;
newZ = motion.x * sin + motion.z * cos;
setDeltaMovement(newX, motion.y, newZ);
```

**掉线机制**：
```
每 tick 计算到发射点距离
  > 16格 → wireGuided = false
         → 发送 "导线断开" 消息给玩家
         → 鱼雷继续直线航行（不追踪、不可控）
```

### 4.3 声导追踪 (`acoustic`)

**原理**：锁定范围内移动/发声的目标，每 tick 限制转弯率追踪。

**参数**：
- 武装延迟：10 ticks
- 正常检测范围：25 格
- 潜行目标检测范围：10 格
- 最大转弯率：8°/tick
- 速度：自动设为 1.0 blocks/tick（声导鱼雷启用时强制降速）

**目标筛选** (`acousticHoming`):
```
扫描 25格范围内 LivingEntity
  过滤：非发射者 && 非友军 && 存活 && 可拾取
  潜行实体 → 检测范围缩减至 10格
  静止+潜行 → 完全不可检测
  选择最近的有效目标
```

**转弯限制**：
```
当前航向角 = atan2(motion.z, motion.x)
目标方位角 = atan2(dz, dx)
差值归一化到 [-π, π]
钳制到 [-8°, +8°]
应用新航向，保持原水平速度
```

## 五、引爆/命中逻辑

### 5.1 命中实体 (`onHitEntity`)

| 鱼雷类型 | 伤害类型 | 附加效果 | 后续 |
|----------|----------|----------|------|
| 普通/线导/声导 | `thrown` | 进水debuff 3秒 (FLOODING, 每秒1点魔法伤害) | discard() |
| 磁性 | `explosion` | 范围爆炸 | magneticDetonate() → discard() |

命中后通知发射者：`[鱼雷名] 命中了 [目标名]` 或 `[鱼雷名] 击杀了 [目标名]`。

鱼雷机投弹时会设置 `sourceAircraftName`，此时通知中武器名显示为飞机名。

### 5.2 命中方块 (`onHitBlock`)

| 方块类型 | 处理 |
|----------|------|
| 水方块 | 静默消失 (`discard()`)，不爆炸 |
| 非水方块（普通鱼雷）| 爆炸 (`explosionRadius`)，discard |
| 非水方块（磁性鱼雷）| `magneticDetonate()`，同上 |

爆炸是否破坏方块由 `ModCommonConfig.EXPLOSION_BLOCK_DAMAGE` 决定。

### 5.3 超时自爆

`lifetime` 每 tick 递减。归零后：
- 在当前位置产生 `explosionRadius` 爆炸
- `discard()`

### 5.4 友军伤害防护

所有碰撞检测均经过 `FriendlyFireHelper.shouldBlockHit(target, owner)` 过滤：
- `canHitEntity()` — 碰撞前检查
- `checkMagneticProximity()` — 近炸过滤
- `acousticHoming()` — 声导目标过滤

玩家发射的鱼雷不会命中发射者自身及其召唤的飞机。

## 六、发射器

### 6.1 发射器型号

| 注册ID | 口径 | 联装数 | 冷却(ticks) | 冷却(秒) |
|--------|------|--------|-------------|---------|
| `twin_torpedo_launcher` | 533 | 2 | 100 | 5.0 |
| `triple_torpedo_launcher` | 533 | 3 | 100 | 5.0 |
| `quad_torpedo_launcher` | 533 | 4 | 100 | 5.0 |

- 所有发射器口径均为 533mm，只能装填 533mm 鱼雷
- 有耐久值，每次齐射消耗 1 点耐久
- 冷却可被装填加速 Buff (`ReloadBoostEffect`) 缩短

### 6.2 装填方式

**装填设施 (`ReloadFacilityBlock`)**：
- 输入：发射器 + 同口径鱼雷
- 加工时间：40 ticks
- 输出：发射器带 `LoadedAmmo` DataComponent
- 每次装满 `tubeCount` 发

**背包自动装填**：
- 需要装备 `TorpedoReloadItem`
- 发射时直接从背包消耗，不需要预装填

### 6.3 手动装填禁用

`TorpedoLauncherItem.overrideOtherStackedOnMe()` 返回 `false`，不允许在物品栏中直接将鱼雷拖入发射器装填。必须使用装填设施方块。

## 七、数据持久化 (NBT)

`TorpedoEntity` 保存/加载以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| Caliber | int | 口径 |
| Damage | float | 伤害 |
| TorpedoSpeed | float | 速度 (读取时 <=0 回退到 1.2) |
| Lifetime | int | 剩余航程ticks (<=0 回退到 1200) |
| ExplosionRadius | float | 爆炸半径 (<=0 回退到 2.0) |
| Magnetic | boolean | 磁性引信 |
| WireGuided | boolean | 线导 |
| Acoustic | boolean | 声导 |
| AirDrop | boolean | 空投阶段 |
| AirDropDirX/Z | double | 空投巡航方向 (仅 airDrop=true 时存) |
| LaunchX/Y/Z | double | 发射点 (仅线导时存) |

## 八、流程总览图

```
┌─────────────────────────────────────────────────────┐
│                    发射源                             │
├──────────┬──────────┬──────────┬─────────────────────┤
│ 玩家手动 │ 玩家背包 │ 鱼雷机   │ 深海NPC             │
│ 装填模式 │ 扫描模式 │ 空投     │ TorpedoAttackGoal   │
└────┬─────┴────┬─────┴────┬─────┴──────────┬──────────┘
     │          │          │                │
     ▼          ▼          ▼                ▼
  new TorpedoEntity(level, player, caliber)  │
  setDamage / setSpeed / setLifetime         │
  setMagnetic / setWireGuided / setAcoustic  │
  setPos + setDeltaMovement                  │
  (空投: airDrop=true, Y=-0.6)              │
     │          │          │         new TorpedoEntity(type, level)
     │          │          │         setOwner(mob) + setDamage(atk)
     │          │          │         setDeltaMovement(fan spread)
     └──────────┴──────────┴────────────────┘
                       │
                       ▼
              TorpedoEntity.tick()
                       │
          ┌────────────┼────────────────┐
          │            │                │
    [空投阶段?]   [正常巡航]       [超时?]
    自由落体      速度恢复          自爆
    着水→巡航     水面贴合
                       │
          ┌────────────┼────────────┐
          │            │            │
     [磁性近炸]   [声导追踪]   [线导控制]
     3格引爆      25格锁定     16格线距
     5tick武装    10tick武装    8°/输入
                       │
          ┌────────────┼────────────┐
          │            │            │
     命中实体      命中方块      超时
     thrown伤害    水中→静默     爆炸

     (磁性→爆炸)
```
