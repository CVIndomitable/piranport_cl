# v0.0.5-alpha — Combat+（战斗强化）

**目标：装填进度 UI 重构（物品装饰器），新增装填加速/高速规避两种自定义战斗 Buff，注册 4 种 Buff 食物及前置食材。**

**设计原则：UI 信息密度优先，Buff 仅变身状态生效，食物忠于原策划案。**

---

## 架构决策

### 装填进度物品装饰器

替换旧的屏幕下方 HUD 装填条，改为在舰装核心物品图标上渲染多条堆叠进度条。

| 决策 | 选择 |
|------|------|
| 技术方案 | `RegisterItemDecorationsEvent` + `IItemDecorator`（NeoForge 标准扩展点） |
| 显示位置 | 舰装核心物品图标（快捷栏中手持的那个） |
| 显示内容 | 所有已装备武器槽的装填进度 |
| 排列方向 | 从上往下，槽 0 在顶部 |
| 旧 HUD 处理 | 保留但默认隐藏（config 可开启） |

**像素布局（16×16 物品图标内）：**

```
┌────────────────┐  ← y=0
│  ▓▓▓▓▓▓▓▓▓▓░░  │  ← y=2  槽0（火炮，橙红色，75%装填）
│                │  ← y=3  间隔
│  ▓▓▓▓▓▓▓▓▓▓▓▓  │  ← y=4  槽1（鱼雷，蓝色，100%装填）
│                │  ← y=5  间隔
│  ░░░░░░░░░░░░  │  ← y=6  槽2（飞机，绿色，0%装填）
│                │  ← y=7~13 空余
│  ████████████  │  ← y=14 原版耐久条（如有）
│                │  ← y=15
└────────────────┘
```

- 每条：1px 高，水平 x=2 到 x=13（宽 12px）
- 每槽占 2px（1px 条体 + 1px 间隔），起始 y=2
- 大型核心 6 槽 = 12px（y=2~y=13），刚好不与底部耐久条冲突
- 未变身时不渲染任何条

**颜色编码：**

| 武器类型 | 装填中 | 装填完成 | 底色（未装填部分） |
|---------|--------|---------|----------------|
| 火炮 | `#E05030` 橙红 | `#FF6A3D` 亮橙 | `#3C3C3C` 深灰 |
| 鱼雷发射器 | `#3070C0` 蓝 | `#50A0FF` 亮蓝 | `#3C3C3C` 深灰 |
| 飞机编队 | `#30A050` 绿 | `#50D070` 亮绿 | `#3C3C3C` 深灰 |
| 空槽 | — | — | 不渲染 |

**数据源：** 客户端通过 `Minecraft.getInstance().player` + 现有 `TransformationManager` 客户端数据读取装填进度，与 `ShipCoreHudLayer` 共用同一数据源，不需要新增 DataComponent 或网络包。

### 自定义战斗 Buff

| Effect | 类别 | 颜色 | 效果 | 限制 |
|--------|------|------|------|------|
| `ReloadBoostEffect` 装填加速 | BENEFICIAL | `#00C8C8` 青色 | 每 tick 装填推进量 = `1 + amplifier`（I 级 2 倍速，II 级 3 倍速） | 仅变身状态下生效 |
| `EvasionEffect` 高速规避 | BENEFICIAL | `#80D0FF` 浅蓝 | 受击时概率完全免伤：I 级 15%，II 级 25%，III 级 35%（封顶） | 仅变身状态下触发 |

**装填加速实现：** `TransformationManager` 装填 tick 逻辑中检查 `player.hasEffect(ModMobEffects.RELOAD_BOOST)`，有则 `progress += 1 + effect.getAmplifier()`。

**高速规避实现：** 监听 `LivingHurtEvent`（`NeoForge.EVENT_BUS`），检查目标是否有 `EvasionEffect` 且处于变身状态，随机判定 `(amplifier + 1) * 0.10 + 0.05` 概率后 `event.setCanceled(true)`。闪避成功时播放 `SoundEvents.WITCH_CELEBRATE`（轻快弹跳音）+ 生成 5 个 `CLOUD` 粒子。

**闪避率公式：**

| 等级 | amplifier | 闪避率 |
|------|-----------|-------|
| I | 0 | 0.05 + 0.10 = 15% |
| II | 1 | 0.05 + 0.20 = 25% |
| III | 2 | 0.05 + 0.30 = 35% |

### Buff 食物

Buff 替换规则：从原策划案食物表中选取，**移除原 Buff，替换为自定义战斗 Buff**，其余属性（饱食度、饱和度、容器、份数）忠于原策划。

**装填加速食物：**

| 食物 | 注册 ID | 饱食度 | 饱和度 | Buff | 持续时间 | 容器 | 份数 | 合成方式 | 配方 |
|------|---------|--------|--------|------|---------|------|------|---------|------|
| 龙田烧 | `chicken_tatsuta` | 6 | 7.5 | 装填加速 I | 180s | 盘 | 2 | 厨锅 | 生鱼/生鲑鱼 + 味噌 + 酱油 + 面粉 + 食用油 |
| 鱼雷果汁 | `torpedo_juice` | 3 | 3.8 | 饥饿 II 180s + 装填加速 II 300s + 抗火 I 300s | — | 玻璃瓶 | 1 | 工作台无序 | 533mm鱼雷 + 菠萝汁 |

> 鱼雷果汁保留原策划的"饥饿+防火"副作用（二战美军用鱼雷乙醇燃料兑果汁的经典梗），核心 Buff 从急迫 III 替换为装填加速 II。
> 鱼雷果汁消耗一枚 533mm 鱼雷（替代原策划的 mk14），成本高昂，对应更强的 Buff 等级和更长持续时间。

**高速规避食物：**

## Phase 实施顺序

### Phase 25: 装填进度物品装饰器

**任务清单：**

1. 新建 `client/ReloadBarDecorator.java`，实现 `IItemDecorator`
   - `render(GuiGraphics, Font, ItemStack, x, y)` 中读取客户端装填数据
   - 遍历武器槽，每个非空槽画一条进度条
   - 条的 y 坐标 = itemY + 2 + slotIndex × 2
   - 条的宽度 = (int)(12 × progress)，progress ∈ [0.0, 1.0]
   - 仅当本地玩家处于变身状态 && 手持物品为 ShipCoreItem 时渲染
2. `ClientEvents` 中监听 `RegisterItemDecorationsEvent`
   - 为三种舰装核心物品注册 `ReloadBarDecorator`
3. `ShipCoreHudLayer` 添加 config 开关
   - 默认 `false`（隐藏），config 设为 `true` 时恢复旧 HUD 条
   - config 使用 NeoForge `ModConfigSpec`（`piranport-client.toml`）
4. 翻译：config 描述 zh_cn + en_us

**验证（构建通过，待 runClient 确认）：**
- [ ] 变身后快捷栏舰装核心图标上出现彩色进度条
- [ ] 条数 = 已装备武器槽数量，颜色按武器类型区分
- [ ] 装填进度实时推进，装填完成后条变亮

- [ ] 大型核心 6 槽满装时条不溢出图标边界

### Phase 26: 自定义战斗 Buff（Effect 注册）

**任务清单：**

1. 新建 `effect/ReloadBoostEffect.java`
   - 继承 `MobEffect`，BENEFICIAL，颜色 `0x00C8C8`
   - Effect 本身不做 tick 逻辑（装填加速在 TransformationManager 中实现）
2. 新建 `effect/EvasionEffect.java`
   - 继承 `MobEffect`，BENEFICIAL，颜色 `0x80D0FF`
   - Effect 本身不做 tick 逻辑（闪避在事件监听中实现）
3. `ModMobEffects.java` 注册两个 Effect
4. `TransformationManager` 装填 tick 修改
   - 检查 `player.hasEffect(ModMobEffects.RELOAD_BOOST)`
   - 有效时 `reloadProgress += 1 + effect.getAmplifier()`（替代固定 +1）
   - 无效时维持原逻辑 `reloadProgress += 1`
5. 新建 `combat/EvasionHandler.java`（或追加到现有 GameEvents）
   - 监听 `LivingHurtEvent`（`NeoForge.EVENT_BUS`）
   - 条件：目标是 ServerPlayer && 有 EvasionEffect && 处于变身状态
   - 闪避判定：`random.nextFloat() < (amplifier + 1) * 0.10 + 0.05`
   - 成功：`event.setCanceled(true)` + 播放音效 + 粒子
   - 粒子和音效需 S2C 包或 `level.playSound` 服务端调用
6. 翻译：`effect.piranport.reload_boost` / `effect.piranport.evasion` zh_cn + en_us

**验证（构建通过，待 runClient 确认）：**
- [ ] 两个 Effect 注册成功，Creative 给予命令可附加
- [ ] 装填加速 I：变身后装填速度明显加快（2 倍）
- [ ] 装填加速 II：装填速度 3 倍
- [ ] 非变身状态下装填加速无实际效果（图标正常显示）
- [ ] 高速规避 I：变身后受击约 15% 概率免伤 + 粒子音效
- [ ] 高速规避 II：约 25% 概率免伤
- [ ] 非变身状态下高速规避不触发闪避
