## 1. 冲突裁决汇总

以下决策来自总策划案、多轮团队讨论稿（2022.2.12、2.12-13、3.1、3.3-3.4、3.17、4.4-5）、受受方案、心宿二方案、航母机制方案之间的冲突裁决。

| # | 冲突点 | 最终决策 | 采纳来源 | 被否决方案 |
|---|--------|---------|----------|-----------|
| 1 | 飞机本质定位 | 飞机 = 武器栏物品，手持右键放飞 | 3.1结论 + 总策划案 | 心宿二（飞机=弹药，甲板=武器）、受受（掉落物回收） |
| 2 | 整备方式 | 燃料自动补充 + 弹药由编组GUI指定 | 混合方案 | 总策划案（GUI拖拽整备）、3.1结论（弹药库顺序） |
| 3 | 侦察机 | v0.4.0 不做，只做战斗机和攻击机 | 新决策 | 总策划案（视角切换）、4.4修订（简化版视角切换） |
| 4 | 火控按键 | 多目标：P锁定/O加选/I取消 | 讨论稿结论 | 总策划案（单目标，自定义按键1/2） |
| 5 | 自动索敌范围 | 半径32格统一 | 总策划案 | 讨论稿（自身16格+控制距离80格） |
| 6 | 编队vs单机 | 编队（单实体多贴图，伤害调整） | 2.12讨论 | 受受/总策划案（纯单机） |
| 7 | 攻击模式分化 | 四种全做：近战/俯冲/鱼雷/水平轰炸 | 总策划案 | 统一投弹、两种分化 |
| 8 | 弹药选择机制 | 编组GUI完全取代弹药库顺序（航空线） | 新决策 | 3.1结论（弹药库顺序） |

## 2. 核心游戏循环

```
武器栏装飞机 → 打开编组GUI分组/指定弹种/攻击模式
→ 变身（自动补燃料、挂易燃易爆debuff）
→ 火控锁定目标（P/O/I）
→ 右键放飞编组（直接右键=顺序、数字键+右键=指定）
→ 飞机AI：索敌→攻击→返航
→ 返航后飞机回武器栏、编组GUI中恢复可编辑
```

## 3. 系统设计

### 3.1 飞机物品

飞机是武器栏物品，与火炮、鱼雷发射器并列，计算负重。

**DataComponent `AircraftInfo`：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `aircraftType` | enum | FIGHTER / DIVE_BOMBER / TORPEDO_BOMBER / LEVEL_BOMBER |
| `fuelCapacity` | int | 燃料容量（决定滞空时间上限） |
| `ammoCapacity` | int | 弹药容量（如天山=1，A2D天鲨=4） |
| `currentFuel` | int | 当前燃料值（0=未整备） |
| `panelDamage` | float | 面板伤害 |
| `panelSpeed` | float | 面板航速 |
| `weight` | int | 负重 |

**DataComponent `AircraftAmmoInfo`：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `assignedAmmoType` | ResourceLocation | 编组GUI指定的弹种（航弹/航空鱼雷/无） |
| `groupIndex` | int | 所属编组（0-3），-1=未编组 |

### 3.2 编组GUI

从舰装栏GUI中的"编组"按钮打开。变身前后均可编辑。

**固定4组**（正航可用4组，轻航可用3组，由舰装核心决定）。

**每组可编辑内容：**

| 属性 | 说明 |
|------|------|
| 成员 | 从武器栏中的飞机拖入/拖出（引用，不移动物品） |
| 弹药指定 | 为本组指定弹种，放飞时从弹药库扣对应弹药，**未指定则不可放飞** |
| 出击顺序 | 组序号即为默认顺序（1→2→3→4），可在GUI中调整 |
| 攻击模式 | 集火（全组打同一个火控目标）/ 分散（各自打最近目标） |

**飞行中状态：** 正在飞行的飞机在编组GUI中变灰、不可编辑，返航后恢复。

**数据存储：** 编组数据存储在舰装核心的 DataComponent 中（`FlightGroupData`），包含4个组的配置。不为每架飞机单独存编组信息，而是核心统一管理。

**DataComponent `FlightGroupData`：**

```
groups: [
  {
    slotIndices: [0, 2],      // 武器栏中的飞机槽位索引
    ammoType: "piranport:aerial_torpedo",
    attackMode: FOCUS,         // FOCUS | SPREAD
    sortOrder: 1
  },
  { ... },
  { ... },
  { ... }
]
```

### 3.3 燃料系统

| 环节 | 行为 |
|------|------|
| 变身时 | 自动从弹药库扣航空燃料，填满武器栏中所有飞机的 `currentFuel`，**弹药库无燃料则该飞机不补** |
| 易燃易爆 | 变身状态 + 武器栏中有 `currentFuel > 0` 的飞机 → 挂 `FlammableEffect`（受伤加成 + 概率额外爆炸伤害） |
| 放飞检查 | `currentFuel == 0` → 拒绝放飞，HUD提示"燃料不足" |
| 返航后 | `currentFuel` 清零 |
| 解除变身 | 不清空燃料（下次变身不重复消耗），但移除易燃易爆debuff |

### 3.4 放飞操作

| 操作 | 行为 |
|------|------|
| 右键 | 按出击顺序放飞下一个未出击的编组 |
| 数字键(1-4) + 右键 | 放飞指定编组 |

放飞时从弹药库扣除编组指定的弹药。弹药不足则提示并拒绝放飞。

放飞动作播放特殊动画（v0.4.0用占位动画，完整版动画留皮肤系统）。飞机实体在玩家前方0.5格生成。

### 3.5 火控系统

**按键绑定（KeyMapping）：**

| 按键 | 功能 |
|------|------|
| P | 准星指向生物实体时按下 → 锁定目标。已有锁定时再按 → 切换到准星方向新目标 |
| O | 加选目标（追加到锁定列表），飞机攻击距自己最近的锁定目标 |
| I | 取消所有锁定 |

**火控数据：** 服务端维护玩家的 `lockedTargets: List<UUID>`，通过 CustomPacketPayload 同步。

**与编组攻击模式的交互：**
- 集火（FOCUS）：本组所有飞机攻击锁定列表中的第一个目标
- 分散（SPREAD）：本组飞机各自攻击距自身最近的锁定目标；无锁定时自动索敌32格内敌对生物

### 3.6 AircraftEntity

**基本属性：**

| 属性 | 说明 |
|------|------|
| `ownerUUID` | 所属玩家 |
| `aircraftType` | FIGHTER / DIVE_BOMBER / TORPEDO_BOMBER / LEVEL_BOMBER |
| `ammoType` | 携带弹药类型 |
| `ammoCount` | 剩余弹药 |
| `fuel` | 剩余燃料（tick递减） |
| `state` | 状态机：LAUNCHING → CRUISING → ATTACKING → RETURNING → REMOVED |
| `targetUUID` | 当前攻击目标 |
| `weaponSlotIndex` | 对应武器栏槽位（用于返航回收） |
| `groupIndex` | 所属编组 |
| `stuckTicks` | 卡死检测计数器 |

**状态机：**

```
LAUNCHING：生成后短暂爬升，持续约1秒
    ↓
CRUISING：飞向目标 / 无目标则32格内索敌 / 无敌人则在玩家附近盘旋
    ↓ (发现目标)
ATTACKING：根据 aircraftType 执行攻击行为
    ↓ (弹药耗尽 / 攻击完成)
RETURNING：飞向玩家位置
    ↓ (到达玩家附近)
REMOVED：实体消失，物品回武器栏
```

**四种攻击行为：**

| 类型 | AI行为 | 伤害方式 |
|------|--------|---------|
| 战斗机 FIGHTER | 接近目标后近战咬尾，每秒造成面板伤害 | 直接近战伤害 |
| 俯冲轰炸机 DIVE_BOMBER | 爬升后向目标俯冲接触，类似幻翼 | 接触时面板伤害 + 50%概率着火30s，弹药值归零后拉起返航 |
| 鱼雷攻击机 TORPEDO_BOMBER | 贴近水面水平飞向目标，距10-16格时投下鱼雷实体 | 复用已有TorpedoEntity的伤害逻辑 |
| 水平轰炸机 LEVEL_BOMBER | 爬升至目标上方32格，水平飞越时投下航弹实体 | 航弹为新投射物，自由落体+接触爆炸 |

**编队表现：** 单实体，但模型和名称显示为"飞行编队"/"飞行中队"。伤害 = 面板伤害 × 编队系数（待数值验证确定）。

### 3.7 防御机制

| 机制 | 触发条件 | 行为 |
|------|---------|------|
| 滞空时间硬上限 | `fuel` 递减到0（硬上限 **10分钟 = 12000 tick**，不论燃料容量） | 强制进入 RETURNING，若已在 RETURNING 则强制 REMOVED + 物品回武器栏 |
| 距离上限 | 与所属玩家距离 > 48格 | TP到玩家附近5格处，状态切为 RETURNING |
| 卡死检测 | 连续60tick（3秒）位置变化 < 0.1格 | 强制 REMOVED + 物品回武器栏 |
| 空手右键回收 | 玩家空手右键自己的飞机实体 | 立即 REMOVED + 物品回武器栏 |
| 玩家死亡 | PlayerDeathEvent | 所有属于该玩家的 AircraftEntity 立即 REMOVED + 物品回武器栏 |
| 玩家下线 | PlayerLoggedOutEvent | 同上 |

### 3.8 浮动靶子（测试工具）

`FloatingTargetEntity`：继承盔甲架的模型和盔甲槽逻辑，增加水面漂浮物理。

| 属性 | 说明 |
|------|------|
| 模型 | 复用原版盔甲架模型 |
| 盔甲 | 可穿戴原版头盔/胸甲/护腿/靴子，获得对应护甲值 |
| 物理 | 放置在水面上时漂浮，不下沉，不被水流推动 |
| 用途 | 航空系统开发期间的伤害测试靶，验证各种攻击模式的伤害数值 |

放置方式：手持浮动靶子物品右键水面生成。

## 4. Phase 拆分

### Phase 18：飞机物品 + 弹药物品 + 编组GUI框架

**内容：**
- 注册飞机物品：战斗机×1型号、俯冲轰炸机×1、鱼雷攻击机×1、水平轰炸机×1
- 注册弹药物品：航空燃料、航弹（小/中）、航空鱼雷
- DataComponent：`AircraftInfo`、`AircraftAmmoInfo`、`FlightGroupData`
- 编组GUI框架：Menu + Screen，从舰装栏按钮打开
- 编组GUI功能：4组显示、飞机拖入拖出、弹药指定下拉、攻击模式切换、出击顺序调整
- 合成表：飞机和弹药的合成（铝锭为主要材料）

**验证：**
- Creative Tab 可见所有新物品
- 编组GUI可从舰装栏打开，可分组、指定弹药、设置攻击模式
- 飞机物品可放入武器栏，计算负重正确

### Phase 19：AircraftEntity + 飞行物理 + 防御机制

**内容：**
- AircraftEntity 注册（继承 Entity，不继承 Mob）
- 状态机：LAUNCHING → CRUISING → ATTACKING → RETURNING → REMOVED
- 飞行物理：爬升、巡航、盘旋、返航的运动计算
- 放飞逻辑：右键 / 数字键+右键 → 检查燃料和弹药 → 生成实体 → 扣弹药
- 返航逻辑：飞向玩家 → 到达后消失 → 物品回武器栏
- 全部防御机制实现：滞空上限、距离TP、卡死检测、右键回收、死亡/下线回收
- 占位渲染器（简单模型或粒子）
- 浮动靶子（FloatingTargetEntity）：盔甲架模型 + 水面漂浮 + 可穿盔甲，用于测试

**验证：**
- 放飞后实体生成、飞行、盘旋
- 滞空时间耗尽后自动返航回收
- 超距TP、卡死检测、手动回收均正常
- 玩家死亡/下线后飞机安全回武器栏

### Phase 20：火控系统

**内容：**
- KeyMapping 注册：P/O/I 三个按键
- 锁定逻辑：准星射线检测生物实体 → 加入/切换/取消锁定列表
- 锁定 HUD：被锁定目标显示标记（菱形/十字）+ 血量/名称
- 网络同步：CustomPacketPayload 同步 lockedTargets 到服务端
- AircraftEntity 读取所属玩家的锁定列表确定目标

**验证：**
- P键锁定目标、O键加选、I键取消
- HUD显示锁定标记
- 放飞后飞机飞向锁定目标

### Phase 21：攻击AI（四种模式）

**内容：**
- 战斗机AI：接近目标 → 咬尾近战 → 持续攻击至目标死亡或燃料耗尽 → 返航
- 俯冲轰炸机AI：爬升 → 俯冲接触 → 伤害+50%着火 → 拉起返航
- 鱼雷攻击机AI：贴水面飞行 → 距目标10-16格投鱼雷（复用TorpedoEntity） → 返航
- 水平轰炸机AI：爬升至目标上方32格 → 计算提前量投弹 → AerialBombEntity（新投射物）→ 返航
- 自动索敌：无锁定目标时在32格内寻找敌对生物
- 编组攻击模式：集火（全组打一个目标）vs 分散（各打最近）
- AerialBombEntity：自由落体、接触后HE爆炸（复用火炮爆炸逻辑）

**验证：**
- 四种飞机分别执行正确的攻击行为
- 战斗机持续攻击、攻击机投弹后返航
- 鱼雷攻击机投下的鱼雷运行正常
- 集火/分散模式行为正确

### Phase 22：燃料系统 + 舰装栏集成 + 数值平衡

**内容：**
- 变身时自动补充燃料逻辑（扣弹药库航空燃料 → 填充飞机 currentFuel）
- 易燃易爆 MobEffect（FlammableEffect）：变身状态+有满油飞机 → 受伤增加
- 舰装栏集成：飞机在武器栏中的负重计算
- 编组弹药消耗：放飞时按编组指定弹种扣弹药库
- 3个典型配装场景数值验证

**数值验证场景（暂定）：**
- 场景1：中型核心 + 2门副炮 + 2架战斗机 + 2架鱼雷机 → 负重合理、输出合理
- 场景2：大型核心 + 4组攻击机满载 → 负重接近上限、DPS验证
- 场景3：小型核心尝试装飞机 → 负重超标被拒或只能装1-2架

**验证：**
- 变身后燃料自动补充，易燃易爆debuff正确挂载
- 无燃料时放飞被拒
- 负重计算正确
- 三个场景全部通过
