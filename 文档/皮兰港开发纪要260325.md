# 皮兰港模组 · 开发纪要

> 文档生成时间：2026-03-25
> 记录范围：从策划案评审到 v0.3.0 规划确定

---

## 1. 原始策划案评审

### 1.1 策划案概况

收到策划文档《总策划案-20220816-更新港区方块.docx》，这是一份 Minecraft 战舰少女R 模组的总策划案（重置版），由"皮兰港"团队编写于 2022 年 7 月，策划案主编 de Nestu，主催轻航独角受。

策划案覆盖了以下子系统：

| 系统 | 完成度 | 复杂度 |
|------|--------|--------|
| 舰装系统（核心、皮肤、负重、变身） | 较高 | 极高 |
| 战斗系统（炮击/雷击/航空/反潜/导弹） | 中等 | 极高 |
| 资源系统（矿物、植物） | 较高 | 低 |
| 方块系统（装饰、功能机器） | 框架 | 中等 |
| 食物-Buff 系统（烹饪、Buff） | 较高 | 中等 |
| NPC 系统（深海、舰娘） | 几乎空白 | 高 |
| 海图系统（副本、深海世界） | 框架 | 极高 |

### 1.2 可行性评估结论

**整体评价：设计雄心大，落地风险高。**

**主要风险：**

1. **范围过大** — 七八个大系统，每个有大量子模块，相当于中型商业游戏的内容量，但团队是同人社团编制。缺少"最小可玩版本"定义。

2. **技术难度被低估** — 侦察机视角切换（留假人模型+强加载区块）、鱼雷/导弹/战斗机 AI、舰装栏 GUI 深度注入原版背包、三层外观渲染系统（皮肤+船体+模块化武器），每一个都是独立的高难度课题。

3. **大量"待补充"** — NPC 系统、海图副本、深海世界维度、剧情/美术/音乐章节基本为空，策划实际完成度约 50-60%。

4. **数值体系未验证** — 负重上限 112/64/40 的设计缺乏典型配装验证案例，"火炮伤害数值设计的目的"标注为"待补充"。

**值得肯定的亮点：**

- 舰装栏和变身系统逻辑完整，锁定/解锁和弹药库惩罚机制考虑周到
- 烹饪系统（锅的热源判定、产物弹出优先级、碗架渲染）细节充分
- 资源系统克制，没有盲目堆砌新矿物
- 版本分阶段推进思路正确

## 2. 技术选型

### 2.1 原方案问题

策划案基于 **Minecraft 1.16.5 + Forge API**，原有工程代码已丢失。

### 2.2 版本选型调研

经调研 2026 年 Minecraft 模组开发生态现状，结论如下：

| 选项 | 状态 | 推荐度 |
|------|------|--------|
| 1.16.5 Forge | 已过时，生态衰退 | ❌ |
| 1.20.1 Forge | 遗留版本，仍有大量模组但不再活跃发展 | ⚠️ |
| 1.21.x NeoForge | 当前主流，16000+ 模组，生态活跃 | ✅ |
| Fabric | 轻量级，适合 Vanilla+ 而非重度内容模组 | ❌ |

**关键事实：**

- 2023 年 Forge 核心团队分裂，主维护者 cpw 等创建 NeoForge
- NeoForge 已成为 1.20.5+ 版本的事实标准，所有主流大型整合包（ATM10、FTB NeoTech 等）均已迁移
- 性能提升显著：启动时间 -32%、内存占用 -19%、TPS +10%
- Mojang 从 26.1 快照开始移除代码混淆，NeoForge 已提前适配

### 2.3 最终选型

```
平台：NeoForge 21.11.x
Minecraft 版本：1.21.11
Java：21
Gradle 插件：ModDevGradle
```

选择 1.21.11 而非 1.21.1 的原因：NeoForge 21.11 是截至决策时的最新稳定版本，API 更成熟，且已有 MDK 模板。

## 3. MVP 实施 (v0.1.0-alpha)

### 3.1 设计思路

从庞大的策划案中裁剪出**一条完整的游戏循环**作为最小可玩版本：

```
挖矿 → 冶炼 → 合成装备 → 装进舰装核心 → 变身 → 开炮打怪
```

### 3.2 MVP 范围（5 个 Phase）

| Phase | 内容 | 验证标准 |
|-------|------|---------|
| Phase 1 | 物品/方块注册、Creative Tab | Creative Tab 可见所有物品 |
| Phase 2 | 矾土矿 + 盐块世界生成 | 新世界能找到矿物 |
| Phase 3 | 合成表 + 熔炉配方 | 工作台/熔炉合成正常 |
| Phase 4 | 舰装核心 GUI（DataComponents + Container） | 右键打开 GUI，放入武器/弹药，显示负重 |
| Phase 5 | 火炮投射物 + 变身状态 | 变身后发射炮弹，HE 爆炸 / AP 穿甲 |

### 3.3 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 物品数据存储 | DataComponents（非旧版 NBT） | 1.21.x 标准做法，Codec 序列化更规范 |
| 世界生成 | Data-Driven JSON + BiomeModifier | 不需要写 Java 注册 Feature |
| 网络同步 | CustomPacketPayload + RegisterPayloadHandlersEvent | NeoForge 21.x 新 API |
| 变身系统 | 简化为 Shift+右键切换 | 原策划的动画/无敌帧/buff清除留到后续 |

### 3.4 MVP 中明确排除的内容

鱼雷、航空、导弹、反潜、皮肤/模型渲染、NPC、海图副本、深海世界、食物/Buff、装饰方块、功能方块、树木作物、剧情系统。

### 3.5 完成状态

**✅ v0.1.0-alpha 已完成。**

## 4. v0.2.0 规划与实施 (Torpedo)

### 4.1 优先级排序

完成 MVP 后，通过评估确定了后续系统的优先级：

| 优先级 | 系统 | 理由 |
|--------|------|------|
| 1 | 鱼雷系统 | 第二武器体系，丰富战斗玩法 |
| 2 | 舰装栏 GUI 完善 + 负重调优 | 核心交互需打磨 |
| 3 | 食物烹饪 + Buff | 后勤体系 |
| 4 | 矿物/资源扩充 + 装饰方块 | 内容填充 |
| 5 | 皮肤/模型渲染 | 美术向，技术复杂，优先级最低 |

### 4.2 v0.2.0 范围（Phase 6-10）

- **Phase 6**: 鱼雷物品（533mm/610mm）+ 发射器（双/三/四联）+ TorpedoEntity 注册
- **Phase 7**: 鱼雷水面贴合物理 + 进水 debuff + 伤害逻辑
- **Phase 8**: 发射器扇形散布 + 弹药消耗 + 冷却/耐久
- **Phase 9**: 舰装栏强化栏 + 变身锁定 + 负重→移速 + 装填 HUD
- **Phase 10**: 数值平衡验证（三个典型配装场景）

### 4.3 完成状态

**✅ v0.2.0-alpha 已完成。**

## 5. v0.3.0 规划 (Kitchen)

### 5.1 系统概述

v0.3.0 实现食物烹饪 + Buff 系统的**基础设施**。策划案中食物表有 60+ 种成品食物，本版本不追求全部注册，而是建好管道（加工站 + 配方系统 + 可放置食物方块框架），注册 ~15 个代表性食物验证全链路。后续版本可零代码改动批量添加食物。

### 5.2 核心游戏循环

```
种植作物 → 收获食材 → 石磨/砧板加工 → 厨锅/工作台/熔炉烹饪 → 手持食用或放置分次食用 → 获得 Buff
```

### 5.3 关键架构决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 可放置食物方块 | 3 种容器形状（盘/碗/蛋糕型）× 1 个通用 BlockEntity | 不为 60+ 食物各注册一个 Block，后续加食物零改动 |
| 配方系统 | 每种加工站一个 RecipeType，JSON 数据驱动 | DataGen 生成，后续加配方不改 Java |
| 酿造 | 复用原版酿造台 + 自定义 IBrewingRecipe | 酿造配方不多，无需自制方块 |
| 石磨加工 | 瞬时加工（放入即产出） | 策划案无明确等待时间，简化交互 |
| 厨锅产物输出 | v0.3.0 手动取出 | 策划案的自动弹出/侧面容器输出留 v0.4.0 |
| 食物 Buff | 全部使用原版 MobEffect | 食物表中无需新增自定义效果 |
| 作物种植 | 包含在 v0.3.0 | 食材获取需要完整种植链路 |

### 5.4 v0.3.0 范围（Phase 11-17）

| Phase | 内容 | 验证标准 |
|-------|------|---------|
| Phase 11 | 食材/调料物品 + 7种新作物种植 | Creative Tab 可见，作物可种植生长收获 |
| Phase 12 | 石磨方块（有GUI，瞬时研磨） | 放入小麦→产出面粉，漏斗兼容 |
| Phase 13 | 砧板方块（无GUI，右键交互） | 放面包→切4次→吐司面包片弹出 |
| Phase 14 | 厨锅方块（有GUI，热源+计时烹饪） | 有热源→放材料→进度推进→产出食物 |
| Phase 15 | 酿造扩展（7个酿造配方） | 水瓶+面粉→酵母瓶 |
| Phase 16 | 可放置食物方块 + ~15个食物注册 | 手持食用+放置+分次食用+碗掉落 |
| Phase 17 | 端到端验证（4个测试场景） | 全链路通过 |

### 5.5 新增内容清单

**新增方块（4 类）：**
- 厨锅（`cooking_pot`）— 有 GUI，热源检测，9 输入 + 1 输出
- 石磨（`stone_mill`）— 有 GUI，4 输入 + 2 输出，瞬时
- 砧板（`cutting_board`）— 无 GUI，右键交互，进度渲染
- 可放置食物（`plate_food` / `bowl_food` / `cake_food`）— 通用容器

**新增作物（7 种）：** 番茄、大豆、辣椒、生菜、稻/米、洋葱、大蒜

**新增食材/调料（22 种）：** 面粉、米粉、辣椒粉、猪肉糜、食用油、黄油、奶油、豆浆、豆腐、奶酪、酵母瓶、酱油、醋、料酒、味噌、盐水、馅饼酥皮、生意面、发酵鱼、披萨饼底、石膏碎片、生石灰

**新增食物（~15 种代表性）：** 吐司面包、海军烘豆子、辣条、麻婆豆腐、海军咖喱、炸鱼薯条、司康饼、咸蛋拌豆腐、鲱鱼罐头、美式汉堡、热狗、意面、米饭、甜豆花、味噌汤

**新增配方系统（3 个 RecipeType）：** `cooking_pot`、`stone_mill`、`cutting_board`

**新增 DataComponent：** `PlaceableInfo`（容器类型 + 食用次数）

### 5.6 v0.3.0 明确排除的内容

- 灶方块（v0.4.0）
- 夕张的水桶（v0.4.0）
- 厨锅自动弹出/侧面容器输出
- 锅内物品/流体/碗架渲染
- 食物方块自定义 3D 模型
- 食物方块右键食材变换（米饭+肉酱=肉酱拌饭）
- 剩余 50+ 种食物注册
- 自定义战斗向 Buff（高速规避、经验提升、装填加速）
- 喂狗功能、流浪商人交易注册
