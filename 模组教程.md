# 皮兰港模组教程

本教程面向想要了解或参与皮兰港模组开发的人，介绍模组的核心玩法与技术实现。

---

## 环境准备

### 开发环境
- **JDK**: Java 21（推荐 Zulu 21）
- **IDE**: IntelliJ IDEA（推荐）
- **构建工具**: Gradle 8.x（Wrapper 已包含，无需单独安装）
- **Minecraft**: 1.21.1
- **模组加载器**: NeoForge 21.1.220

### 首次构建
```bash
git clone https://github.com/CVIndomitable/piranport_cl.git
cd piranport_cl
./gradlew build
```

构建产物位于 `build/libs/piranport-0.0.3.jar`。

### 导入 IDE
```bash
./gradlew genIntellijRuns   # 生成 IntelliJ 运行配置
```
然后在 IntelliJ 中 File → Open 选择项目根目录。

### 运行客户端
```bash
./gradlew runClient
```

## 模组功能概览

### 1. 舰装系统

舰娘核心机制。手持**舰装核心**（小/中/大型）Shift+右键变身：

- 变身后进入舰装模式，右键打开舰装栏 GUI
- **武器栏**（左侧）：装备火炮或鱼雷发射器
- **弹药库**（中间）：存放对应炮弹/鱼雷
- **强化栏**（右侧）：装备附加装甲（+2/+4/+6防御）
- **负重系统**：武器+弹药+装甲总重量不超过核心上限，超重无法装备
- **移速**：负重越接近上限，移速越慢（最低约40%）

#### 武器切换
`Tab`（默认）切换当前选中武器槽，右键发射当前武器。

### 2. 火炮战斗

| 武器 | 弹药 | 特性 |
|------|------|------|
| 小型火炮 | 小口径 HE/AP | 装填快，伤害低 |
| 中型火炮 | 中口径 HE/AP | 均衡 |
| 大型火炮 | 大口径 HE/AP | 伤害高，装填慢 |

- **HE炮弹**：爆炸伤害，范围溅射
- **AP炮弹**：穿甲，对高护甲目标更有效

### 3. 鱼雷系统

| 物品 | 说明 |
|------|------|
| 533mm标准鱼雷 | 基础鱼雷弹药 |
| 610mm氧气鱼雷 | 高伤害鱼雷弹药 |
| 双联装鱼雷发射器 | 2枚，±3°散布 |
| 三联装鱼雷发射器 | 3枚，±4°散布 |
| 四联装鱼雷发射器 | 4枚，±6°/±2°散布 |

- 鱼雷在水面贴合航行，命中目标施加**进水**Debuff（每秒魔法伤害）
- 飞行中遇水自动进入水面模式

## 开发者指南

### 添加新食物（只需3步）

**Step 1**: 在 `ModItems.java` 注册物品

```java
public static final DeferredItem<Item> MY_FOOD = ITEMS.register("my_food",
    () -> new ModFoodItem(new Item.Properties()
        .food(fp(5, 6.3f)  // nutrition=5, 饱和度=6.3
            .effect(() -> new MobEffectInstance(MobEffects.MOVEMENT_SPEED, 200, 1), 1.0f)
            .build())
        .component(ModDataComponents.PLACEABLE_INFO.get(), new PlaceableInfo("plate", 2))));
```

**Step 2**: 写一条 JSON 配方（厨锅示例）

`src/main/resources/data/piranport/recipe/cooking_pot_my_food.json`
```json
{
  "type": "piranport:cooking_pot",
  "ingredients": [
    { "item": "minecraft:wheat" },
    { "item": "piranport:salt" }
  ],
  "result": { "id": "piranport:my_food", "count": 1 },
  "cookingTime": 200
}
```

**Step 3**: 添加翻译

`zh_cn.json`: `"item.piranport.my_food": "我的食物"`
`en_us.json`: `"item.piranport.my_food": "My Food"`

完成！无需修改任何 Java 逻辑代码。

### 注册系统规范

所有注册走 `DeferredRegister`，在主类 `PiranPort.java` 构造函数中 `.register(modEventBus)`。

| 类型 | 注册类 |
|------|--------|
| 物品 | `ModItems.ITEMS` |
| 方块 | `ModBlocks.BLOCKS` |
| 方块实体类型 | `ModBlockEntityTypes.BLOCK_ENTITY_TYPES` |
| 菜单类型 | `ModMenuTypes.MENU_TYPES` |
| 配方类型+序列化器 | `ModRecipeTypes.RECIPE_TYPES` / `RECIPE_SERIALIZERS` |
| DataComponent | `ModDataComponents.DATA_COMPONENTS` |

### 构建与测试

```bash
./gradlew build          # 构建 JAR
./gradlew runClient      # 启动游戏客户端
./gradlew runData        # 运行 DataGen（生成 JSON）
```

构建成功后 JAR 自动复制到 Minecraft mods 目录（Stop hook 配置）。

## 常见问题

**Q: 变身后无法使用武器？**
A: 检查弹药库是否有对应弹药，负重是否超出核心上限。

**Q: 厨锅不烹饪？**
A: 确认锅正下方有热源（营火需点燃，熔炉需在燃烧状态）。

**Q: 酿造台没有配方？**
A: 水瓶放在底部三槽，试剂放在顶部槽，确保使用原版酿造台。

**Q: 食物放置后不显示？**
A: 渲染基于 BlockEntityRenderer，需要客户端加载完毕。
