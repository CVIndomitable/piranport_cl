# 皮兰港 Mod 高光效果排查报告

生成时间：2026-05-09

## 一、高光系统架构

### 1.1 优先级设计（从高到低）

根据记忆 `feedback_glow_priority.md`：
```
原版高光 > 玩法高光(火控) > Y键战场高亮
```

### 1.2 三种高光机制

#### **机制1：原版高光（最高优先级）**
- **触发条件**：发光效果（Glowing MobEffect）、光灵箭
- **实现位置**：`Entity.isCurrentlyGlowing()` 原版方法
- **检测方法**：`ClientTickHandler.hasVanillaGlow(Entity)` (line 73-78)
- **颜色**：由原版 scoreboard team 决定
- **保护机制**：所有 mod 高光逻辑都会检查 `hasVanillaGlow()`，确保不移除原版高光

#### **机制2：火控锁定高光（第二优先级）**
- **触发条件**：P键锁定、O键添加目标
- **管理类**：`ClientFireControlData`（客户端）、`FireControlManager`（服务端）
- **实现位置**：`ClientTickHandler.onClientTick()` (line 290-329)
- **颜色**：红色（`ChatFormatting.RED`）
- **Scoreboard Team**：`pp_fc_target`
- **特殊处理**：
  - `AircraftEntity` 不加入 FC team（line 311-313），通过 `getTeamColor()` 返回红色
  - 其他实体通过 scoreboard team 显示红色描边

#### **机制3：Y键战场高亮（最低优先级）**
- **触发条件**：按Y键启用 `highlightEnabled`
- **实现位置**：`ClientTickHandler.onClientTick()` (line 266-287, 290-329)
- **目标选择**：`isHighlightTarget()` (line 368-393)
  - 玩家自己的子弹（`BulletEntity`）
  - 敌对生物（32格内）
  - 怪物（32格内）
  - 动物（16格内）
- **颜色**：白色（默认）
- **性能优化**：每4 tick扫描一次（line 300-301, 323）

#### **机制4：ASW声呐高光（独立系统）**
- **触发条件**：反潜机声呐扫描到水下目标
- **管理类**：`ClientAswSonarData`
- **实现位置**：`ClientTickHandler.onClientTick()` (line 332-361)
- **颜色**：黄色（`ChatFormatting.YELLOW`）
- **Scoreboard Team**：`pp_asw_sonar`
- **优先级处理**：FC目标不加入ASW team（line 345-347）

## 三、冲突点分析

### 3.1 ⚠️ 冲突1：地牢实体强制高光

**问题实体**：
- `LowTierDestroyerEntity` (line 171-173)
- `DungeonTransportPlaneEntity` (line 159-161)

**冲突原因**：
```java
@Override
public boolean isCurrentlyGlowing() {
    return true;  // 永远返回true，无视任何优先级
}
```

**影响**：
1. 这些实体**无法被火控锁定改变颜色**（因为 `isCurrentlyGlowing()` 不检查火控状态）
2. 这些实体**无法显示ASW声呐黄色**（同理）
3. 违反了"原版 > 火控 > Y键"的优先级设计

**修复建议**：
```java
@Override
public boolean isCurrentlyGlowing() {
    // 地牢实体默认发光，但尊重原版高光优先级
    if (super.isCurrentlyGlowing()) return true;
    return true;  // 地牢实体始终发光
}
```

### 3.2 ⚠️ 冲突2：AircraftEntity 颜色优先级

**问题位置**：`AircraftEntity.getTeamColor()` (line 2150-2159)

**当前逻辑**：
```java
@Override
public int getTeamColor() {
    if (level().isClientSide()) {
        // 原版高光优先
        if (super.isCurrentlyGlowing() && !AircraftGlowHelper.isFcTarget(this)) {
            return super.getTeamColor();
        }
        return AircraftGlowHelper.getGlowColor(this);
    }
    return super.getTeamColor();
}
```

**潜在问题**：
- 当飞机**同时**被火控锁定 + 有原版发光效果时，会显示**火控红色**而非原版颜色
- 违反了"原版 > 火控"的优先级

**修复建议**：
```java
@Override
public int getTeamColor() {
    if (level().isClientSide()) {
        // 原版高光优先（无条件）
        if (super.isCurrentlyGlowing()) {
            return super.getTeamColor();
        }
        return AircraftGlowHelper.getGlowColor(this);
    }
    return super.getTeamColor();
}
```

### 3.4 ⚠️ 冲突3：ASW声呐与火控的team冲突

**问题位置**：ClientTickHandler line 345-347

**当前逻辑**：
```java
// FC team takes priority — don't add to ASW team if already FC target
if (!lockedTargets2.contains(entity.getUUID())) {
    currentAswMembers.add(entity.getStringUUID());
}
```

**问题**：
- 当实体**先被ASW标记（黄色）**，**后被火控锁定（红色）**时：
  1. 实体会被加入 `pp_fc_target` team（红色）
  2. 但**不会从** `pp_asw_sonar` team 移除
  3. 导致实体**同时在两个team中**

**Minecraft行为**：
- 一个实体只能在一个team中
- 后加入的team会覆盖前一个team
- 但 `aswTeamMembers` 集合中仍保留该实体，导致状态不一致

**修复建议**：
在 `syncFcTeam()` 中添加：
```java
// 从ASW team移除FC目标
for (String name : currentMembers) {
    if (aswTeamMembers.contains(name)) {
        scoreboard.removePlayerFromTeam(name, aswTeam);
        aswTeamMembers.remove(name);
    }
}
```

### 修复2：syncFcTeam() 添加ASW清理

```java
private static void syncFcTeam(Minecraft mc, Set<String> currentMembers) {
    if (mc.level == null) return;
    net.minecraft.world.scores.Scoreboard scoreboard = mc.level.getScoreboard();
    net.minecraft.world.scores.PlayerTeam team = scoreboard.getPlayerTeam(FC_TEAM_NAME);
    net.minecraft.world.scores.PlayerTeam aswTeam = scoreboard.getPlayerTeam(ASW_TEAM_NAME);
    
    if (team == null && !currentMembers.isEmpty()) {
        team = scoreboard.addPlayerTeam(FC_TEAM_NAME);
        team.setColor(net.minecraft.ChatFormatting.RED);
    }
    if (team == null) {
        fcTeamMembers.clear();
        return;
    }
    
    // Remove members no longer targeted
    for (String name : new HashSet<>(fcTeamMembers)) {
        if (!currentMembers.contains(name)) {
            scoreboard.removePlayerFromTeam(name, team);
        }
    }
    
    // Add new members and remove from ASW team (FC has higher priority)
    for (String name : currentMembers) {
        if (!fcTeamMembers.contains(name)) {
            // 从ASW team移除（如果存在）
            if (aswTeam != null && aswTeamMembers.contains(name)) {
                scoreboard.removePlayerFromTeam(name, aswTeam);
                aswTeamMembers.remove(name);
                aswHighlightedEntityIds.removeIf(id -> {
                    Entity e = mc.level.getEntity(id);
                    return e != null && e.getStringUUID().equals(name);
                });
            }
            scoreboard.addPlayerToTeam(name, team);
        }
    }
    
    fcTeamMembers.clear();
    fcTeamMembers.addAll(currentMembers);
}
```

## 十、结论

当前高光系统整体架构清晰，但存在**3个优先级冲突**需要修复：

1. **P0级**：`AircraftEntity` 颜色优先级错误
2. **P0级**：ASW与火控的team切换不完整
3. **P1级**：地牢实体无法响应火控

修复后，系统将完全符合"原版 > 火控 > Y键"的优先级设计。
